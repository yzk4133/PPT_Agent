# 亮点三：统一工具系统（LangChain Tools + 技能系统）

## 版本一：完整版（1-1.5 分钟）

项目初期每个 Agent 各自管理自己的工具，导致接口不统一、难以维护。比如搜索工具、数据库工具的调用方式都不一样，添加新工具需要修改多个地方。

所以我搭建了一个**统一的工具注册中心**，整合了所有 LangChain Tools 和自定义 Skills。核心是一个 **NativeToolRegistry**，系统启动时自动扫描 `domain/` 和 `skills/` 目录，自动注册所有工具到注册中心。

工具按功能分为 6 大类：搜索类、媒体类、数据库类、工具类、向量检索类、技能类。目前整合了 **20 多个工具**，包括网络搜索、微信文章搜索、图片搜索、数据库操作、PPT 生成等。

每个工具都继承 LangChain 的 `BaseTool`，使用 **Pydantic** 定义参数 Schema 实现自动校验。Agent 可以通过注册中心**动态获取**需要的工具，支持按需调用，不需要在初始化时加载所有工具。

我还实现了一套**技能系统**，支持两种类型：Python Skills（可执行的工作流程）和 MD Skills（提示词模板）。这些技能会自动转换为 LangChain Tools，可以和其他工具一样被 Agent 调用。

对于少数需要独立部署的工具（如 web_search），我通过 **MCP 协议**实现了可选的 MCP Server 版本，可以通过环境变量控制是否启用，MCP 不可用时会自动降级到 LangChain 版本，保证系统稳定性。

---

## 版本二：简化版（1 分钟内）

为了解决工具接口不统一的问题，我搭建了**统一工具注册中心**，整合了 **20 多个 LangChain Tools**。

工具按功能分为 6 大类（搜索、媒体、数据库、工具、向量检索、技能），系统启动时**自动扫描和注册**。每个工具有统一的调用接口和参数校验，Agent 可以**动态获取**需要的工具。

还实现了**技能系统**，支持 Python Skills（工作流程）和 MD Skills（提示词模板），自动转换为 LangChain Tools。少数工具支持 **MCP 协议**独立部署，可根据需要启用。

添加新工具只需要实现 LangChain `BaseTool` 接口，注册后所有 Agent 都能使用。

```python
research_workflow_tool = StructuredTool.from_function(
```

```python
    registry.register_tool(web_search_mcp_tool, category="SEARCH")
```

- 灵活性：需要独立部署时可以启用
- 安全性：敏感工具可以隔离运行
- 跨语言：可以用其他语言实现 MCP Server
- 可选性：不影响主流程，MCP 不可用时自动降级

### Q8: 如何添加新工具？

**简短回答**：实现 LangChain BaseTool 接口，放到对应目录，自动注册。

**详细回答**：

**步骤 1：实现工具**

```python
# backend/tools/domain/custom/my_tool.py

from langchain_core.tools import BaseTool
from langchain_core.pydantic_v1 import BaseModel, Field
from typing import Optional, Type

class MyToolInput(BaseModel):
    """输入参数"""
    param1: str = Field(description="参数1")
    param2: int = Field(default=10, ge=1, le=100)

class MyTool(BaseTool):
    """我的自定义工具"""
    name = "my_tool"
    description = "工具描述"
    args_schema: Type[BaseModel] = MyToolInput

    def _run(self, param1: str, param2: int = 10) -> str:
        """同步执行"""
        result = f"Processed {param1} with {param2}"
        return result

    async def _arun(self, param1: str, param2: int = 10) -> str:
        """异步执行"""
        return self._run(param1, param2)
```

**步骤 2：注册工具**

```python

from backend.tools.domain.custom.my_tool import MyTool

# 创建实例并注册

```

**步骤 3：使用工具**

```python

    "param1": "value1",
    "param2": 20
```

**自动注册（推荐）**：

```python
# 修改工具注册逻辑，支持自动扫描

def _auto_register_tools():
    """自动注册所有工具"""

    # 扫描 domain/ 目录
    domain_dir = Path("backend/tools/domain")
    for category_dir in domain_dir.iterdir():
        if category_dir.is_dir():
            for tool_file in category_dir.glob("*_tool.py"):
                module = import_module(f"backend.tools.domain.{category_dir.name}.{tool_file.stem}")

                # 查找 BaseTool 子类
                for name in dir(module):
                    cls = getattr(module, name)
                    if isinstance(cls, type) and issubclass(cls, BaseTool) and cls != BaseTool:
                        tool = cls()
                        _global_registry.register_tool(tool, category=category_dir.name.upper())
```

### Q9: 工具调用失败如何处理？

**简短回答**：重试、降级、跳过三种策略。

**详细回答**：

**策略 1：重试**

```python
async def call_tool_with_retry(tool, params, max_retries=3):
    """带重试的工具调用"""
    for attempt in range(max_retries):
        try:
            result = await tool.ainvoke(params)
            return result
        except (TimeoutError, ConnectionError) as e:
            if attempt == max_retries - 1:
                raise
            logger.warning(f"Retry {attempt+1}/{max_retries}: {e}")
            await asyncio.sleep(2 ** attempt)
```

**策略 2：降级**

```python
async def call_tool_with_fallback(primary_name, fallback_name, params):
    """带降级的工具调用"""
    registry = get_native_registry()

    # 尝试主工具
    primary_tool = registry.get_tool(primary_name)
    try:
        return await primary_tool.ainvoke(params)
    except Exception as e:
        logger.warning(f"Primary tool {primary_name} failed: {e}")

        # 降级到备用工具
        fallback_tool = registry.get_tool(fallback_name)
        return await fallback_tool.ainvoke(params)

# 示例：MCP 版本失败时降级到 LangChain 版本
result = await call_tool_with_fallback(
    primary_name="web_search_mcp",
    fallback_name="web_search",
    params={"query": "AI", "num_results": 5}
)
```

**策略 3：跳过**

```python
async def call_tool_optional(tool_name, params):
    """可选的工具调用，失败时跳过"""
    registry = get_native_registry()
    tool = registry.get_tool(tool_name)

    if tool is None:
        logger.warning(f"Tool {tool_name} not available, skipping")
        return None

    try:
        return await tool.ainvoke(params)
    except Exception as e:
        logger.warning(f"Tool {tool_name} failed, skipping: {e}")
        return None
```
