# 亮点四：分层记忆系统（MySQL + Redis）

## 版本一：完整版（1-1.5 分钟）

为了实现个性化的用户体验，系统需要记住用户的偏好和历史内容。但直接在 Agent 中维护状态会导致数据丢失、无法跨会话共享、性能差等问题。

所以我设计了一套**分层记忆系统**，采用 **MySQL + Redis** 的三层架构：进程内内存、Redis L2 缓存、MySQL 持久化。进程内内存存储当前会话状态，Redis 缓存热点数据（如用户偏好），MySQL 持久化所有重要数据。

Agent 通过继承 `MemoryAwareAgent` 基类获得记忆能力，使用简单的接口如 `remember()`、`recall()` 就能存储和检索记忆。系统会自动从 LangGraph 状态中提取上下文（task_id、user_id、session_id），并根据键名自动推断记忆的作用域。

读取时先查 Redis 缓存，未命中再查 MySQL，并将结果回写到 Redis。写入时同时更新 MySQL 和 Redis，保证数据一致性。如果 Redis 不可用，系统会自动降级到直接使用 MySQL，保证稳定性。

目前系统主要管理用户偏好（语言、风格、颜色等），缓存命中率达到 **85%**，用户再次使用时几乎感觉不到延迟。

## 可能的追问

### Q1: 为什么要用三层架构，而不是直接用 MySQL？

**简短回答**：平衡性能和成本，每层有不同职责。

**详细回答**：

**三层对比**：

| 层 | 存储介质 | 速度 | 容量 | 持久性 | 职责 |
|----|---------|------|------|--------|------|
| **L1（进程内）** | Python dict | 最快（<1ms） | 最小 | 否 | 当前会话状态 |
| **L2（Redis）** | 内存缓存 | 快（<5ms） | 中等 | 否 | 热点数据、用户偏好 |
| **L3（MySQL）** | 磁盘数据库 | 慢（~50ms） | 大 | 是 | 持久化存储 |

**为什么需要三层**：

```
只用 MySQL：每次都查库，延迟高、压力大
只用 Redis：数据易丢失、容量受限
三层架构：热数据走缓存，冷数据走持久化，兼顾性能和可靠性
```

**数据分布示例**：

```
L1（进程内）:
  - 当前生成任务的 state
  - Agent 临时变量
  - ~100 条记录，会话结束清空

L2（Redis）:
  - 用户偏好（user_pref:{user_id}）
  - 最近研究结果
  - 会话缓存
  - ~10K 条记录，TTL 24h

L3（MySQL）:
  - 用户配置表
  - 决策历史
  - 完整会话记录
  - ~100 万条记录，永久保留
```

### Q2: 缓存策略是怎样的？如何保证一致性？

**简短回答**：读时回写、写时同步、失效策略、降级方案。

**详细回答**：

**读取流程**：

```python
async def recall(key, scope=None):
    """检索记忆"""

    # 1. 检查 L1（进程内）
    if key in self._l1_cache:
        return self._l1_cache[key]

    # 2. 检查 L2（Redis）
    cached = await self.redis.get(key)
    if cached:
        # 回写到 L1
        self._l1_cache[key] = cached
        return cached

    # 3. 查询 L3（MySQL）
    result = await self.mysql.get(key)
    if result:
        # 回写到 L2
        await self.redis.set(key, result, ttl=3600)
        # 回写到 L1
        self._l1_cache[key] = result

    return result
```

**写入流程**：

```python
async def remember(key, value, scope=None):
    """存储记忆"""

    # 1. 写入 L3（MySQL）
    await self.mysql.set(key, value)

    # 2. 写入 L2（Redis），带 TTL
    await self.redis.set(key, value, ttl=3600)

    # 3. 写入 L1（进程内）
    self._l1_cache[key] = value

    return True
```

**一致性保证**：

| 策略 | 实现方式 | 说明 |
|------|----------|------|
| **写穿透** | 写 MySQL 时同步写 Redis | 保证数据一致 |
| **失效策略** | 用户更新偏好时删除缓存 | 避免脏读 |
| **版本号** | 每条记录有 version 字段 | 冲突时检测 |
| **最终一致性** | 允许短暂不一致 | 通过后台任务同步 |

**降级方案**：

```python
# Redis 不可用时的降级
if not self.redis.is_available():
    # 直接查询 MySQL
    result = await self.mysql.get(key)
    return result
```

### Q4: 作用域推断是如何实现的？

**简短回答**：根据键名关键词自动推断记忆作用域（TASK、USER、SESSION）。

**详细回答**：

**作用域定义**：

```python
class MemoryScope(Enum):
    """记忆作用域"""
    TASK = "TASK"        # 任务级，仅当前任务可见
    SESSION = "SESSION"  # 会话级，跨任务共享
    USER = "USER"        # 用户级，跨会话持久化
    WORKSPACE = "WORKSPACE"  # 工作空间，跨 Agent 共享
```

**推断规则**：

```python
def _infer_scope_from_key(self, key: str) -> str:
    """根据键推断作用域"""
    key_lower = key.lower()

    # 状态相关 → TASK
    if any(kw in key_lower for kw in
           ["state", "status", "progress", "current"]):
        return "TASK"

    # 用户数据 → USER
    elif any(kw in key_lower for kw in
             ["preference", "config", "setting", "profile"]):
        return "USER"

    # 共享数据 → WORKSPACE
    elif any(kw in key_lower for kw in
             ["shared", "workspace", "common"]):
        return "WORKSPACE"

    # 默认 → TASK
    else:
        return "TASK"
```

**示例**：

| 键名 | 推断作用域 | 说明 |
|------|-----------|------|
| `research_result` | TASK | 当前任务的研究结果 |
| `user_language` | USER | 用户语言偏好 |
| `shared_context` | WORKSPACE | Agent 间共享上下文 |
| `generation_progress` | TASK | 生成进度 |

**作用域 ID 映射**：

```python
def _get_scope_id(self, scope: str) -> str:
    """根据作用域返回对应的 ID"""
    if scope == "USER":
        return self._user_id or "anonymous"
    elif scope == "WORKSPACE":
        return self._session_id or self._task_id or "default"
    else:  # TASK, SESSION
        return self._task_id or self._session_id or "default"
```

```python
class RedisCache:
    """Redis 缓存（带降级）"""

    def __init__(self, redis_url):
        try:
            self.client = redis.from_url(redis_url)
            self.client.ping()
            self.available = True
        except:
            self.available = False
            logger.warning("Redis unavailable, will use DB only")

    async def get(self, key):
        """获取缓存（降级处理）"""
        if not self.available:
            return None  # 降级：直接返回 None

        try:
            value = await self.client.get(key)
            return json.loads(value)
        except:
            return None
```

### Q6: 如何监控记忆系统的性能？

**简短回答**：缓存命中率、延迟监控、统计信息。

**详细回答**：

**缓存命中率统计**：

```python
class MemorySystem:
    def __init__(self):
        self.stats = {
            "l1_hits": 0,
            "l2_hits": 0,
            "l3_hits": 0,
            "misses": 0
        }

    async def recall(self, key):
        """检索记忆（带统计）"""
        # L1 命中
        if key in self._l1_cache:
            self.stats["l1_hits"] += 1
            return self._l1_cache[key]

        # L2 命中
        cached = await self.redis.get(key)
        if cached:
            self.stats["l2_hits"] += 1
            self._l1_cache[key] = cached
            return cached

        # L3 命中
        result = await self.mysql.get(key)
        if result:
            self.stats["l3_hits"] += 1
            return result

        # 未命中
        self.stats["misses"] += 1
        return None

    def get_cache_hit_rate(self):
        """计算缓存命中率"""
        total = sum(self.stats.values())
        if total == 0:
            return 0

        hits = self.stats["l1_hits"] + self.stats["l2_hits"]
        return hits / total
```

**性能指标**：

```python
def get_statistics(self):
    """获取统计信息"""
    total = sum(self.stats.values())

    return {
        "total_requests": total,
        "cache_hit_rate": self.get_cache_hit_rate(),
        "l1_hit_rate": self.stats["l1_hits"] / total if total > 0 else 0,
        "l2_hit_rate": self.stats["l2_hits"] / total if total > 0 else 0,
        "l3_hit_rate": self.stats["l3_hits"] / total if total > 0 else 0,
        "miss_rate": self.stats["misses"] / total if total > 0 else 0,
    }
```

```python
# backend/memory/services/custom_memory_service.py

class CustomMemoryService(BaseService):
    """自定义记忆服务"""

    async def save_memory(self, user_id, memory_type, content):
        """保存记忆"""
        memory = CustomMemory(
            user_id=user_id,
            memory_type=memory_type,
            content=content
        )

        self.db_session.add(memory)
        self.db_session.commit()
        self.db_session.refresh(memory)

        return memory.to_dict()

    async def get_memories(self, user_id, memory_type=None, limit=10):
        """获取记忆"""
        query = self.db_session.query(CustomMemory)\
            .filter_by(user_id=user_id)

        if memory_type:
            query = query.filter_by(memory_type=memory_type)

        memories = query.order_by(CustomMemory.created_at.desc())\
            .limit(limit)\
            .all()

        return [m.to_dict() for m in memories]
```

### Q8: 记忆系统和 LangGraph 状态如何协同？

**简短回答**：从 state 提取上下文、记忆存储到 state、不影响 state 传递。

**详细回答**：

**上下文提取**：

```python
def _get_memory(self, state):
    """从 LangGraph 状态初始化记忆"""

    # 提取上下文
    self._task_id = state.get("task_id")
    self._user_id = state.get("user_id", "anonymous")
    self._session_id = state.get("session_id", self._task_id)

    # 创建记忆管理器
    self._memory = StateBoundMemoryManager(
        state=state,
        agent_name=self.agent_name,
        task_id=self._task_id,
        user_id=self._user_id,
        session_id=self._session_id
    )

    return self._memory
```

**记忆存储到 state**（可选）：

```python
async def run_node(self, state):
    # 1. 初始化记忆
    self._get_memory(state)

    # 2. 使用记忆
    cached = await self.recall("research_result")

    # 3. 执行业务逻辑
    result = cached or await self.do_research(state["query"])

    # 4. 存储记忆
    await self.remember("research_result", result)

    # 5. 可选：将记忆存储到 state（供下游 Agent 使用）
    state["research_results"] = result

    return state  # LangGraph 继续传递 state
```

**不影响状态传递**：

```python
# 记忆系统独立于 state
# state 只在 Agent 之间传递
# 记忆通过记忆系统管理

# → 使用记忆系统存储结果
```

### Q9: 如何处理并发访问和竞态条件？

**简短回答**：Redis 分布式锁、数据库事务、版本控制。

**详细回答**：

**Redis 分布式锁**：

```python
async def update_preferences_with_lock(self, user_id, updates):
    """带锁的偏好更新"""

    lock_key = f"lock:user_pref:{user_id}"
    lock_value = str(uuid.uuid4())

    try:
        # 获取锁（30秒超时）
        acquired = await self.redis.set(
            lock_key,
            lock_value,
            nx=True,  # 仅当 key 不存在时设置
            ex=30     # 30秒自动过期
        )

        if not acquired:
            raise Exception("Failed to acquire lock")

        # 执行更新
        profile = await self.get(UserProfile, user_id)
        for key, value in updates.items():
            profile.set_preference(key, value)

        self.db_session.commit()

    finally:
        # 释放锁（Lua 脚本保证原子性）
        await self.redis.eval(
            "if redis.call('get', KEYS[1]) == ARGV[1] then "
            "return redis.call('del', KEYS[1]) "
            "else return 0 end",
            1, lock_key, lock_value
        )
```

**数据库事务**：

```python
async def create_with_transaction(self, data_list):
    """事务中创建多条记录"""

    try:
        for data in data_list:
            instance = CustomMemory(**data)
            self.db_session.add(instance)

        # 事务提交
        self.db_session.commit()

    except Exception as e:
        # 回滚
        self.db_session.rollback()
        raise
```

**版本控制**：

```python
class UserProfile(BaseModel):
```
