# PPT生成多智能体（1主5子）架构设计文档

## 文档概述

本文档详细阐述PPT自动生成系统的多智能体架构设计，核心采用「1个主协调智能体+5个子功能智能体」的极简高效架构（以下简称「1主5子」架构）。该架构基于「复用PPT模板」的核心前提，砍掉冗余排版相关智能体，新增资料研究智能体，兼顾「功能完整性、落地可行性、用户体验」，既解决单智能体能力过载、内容单薄的痛点，又适配「前端实时预览、最终导出可编辑PPT」的交互需求，同时便于面试中清晰讲透分工、协作逻辑与设计亮点。

架构设计遵循「单一职责、低耦合、高内聚」原则，所有子智能体仅与主协调智能体交互，无直接跨智能体通信，由主智能体统一把控任务调度、信息流转与结果校验，形成「需求输入→研究支撑→内容创作→模板渲染→前端展示→导出落地」的完整闭环。

## 一、架构总览

### 1.1 架构核心定位

以「轻量落地、专业高效、用户友好」为核心，通过多智能体分工协作，将PPT生成流程拆解为可量化、可反馈的阶段任务，实现「一边生成、一边前端预览、最终支持导出」的核心需求，同时通过资料研究智能体提升PPT内容的专业性与说服力，摆脱单纯的内容拼接模式。

### 1.2 整体架构图（文字描述，面试可直接口述）

用户需求 → 【主协调智能体】（中枢调度）→ 子智能体执行（按阶段顺序）→ 前端实时预览 → 导出可编辑PPT

子智能体执行顺序：需求解析智能体 → 框架设计智能体 → 资料研究智能体 → 内容素材智能体 → PPT模板渲染智能体

### 1.3 架构核心优势

1.  极简高效：仅6个智能体（1主5子），砍掉冗余排版环节，基于PPT模板复用，落地成本低、生成效率高；

2.  专业升级：新增资料研究智能体，为内容提供行业数据、案例、权威来源支撑，提升PPT说服力；

3.  低耦合高内聚：子智能体仅与主智能体交互，新增/修改功能仅需调整对应智能体，拓展性强；

4.  体验友好：适配前端异步分段预览，用户可实时查看生成进度与内容，支持修订功能，无需从头生成；

5.  面试适配：各智能体职责清晰、输入输出标准化，可清晰讲透「分工逻辑、联动机制、痛点解决」，突出设计思维与工程落地能力。

## 二、智能体详细拆解（核心章节）

所有智能体均采用「标准化输入/输出（JSON格式）+ 单一核心职责 + 自校验机制」设计，确保每个智能体可独立执行、可单独调试，同时便于主智能体统一调度与校验。

### 2.1 核心中枢：主协调智能体（架构大脑，面试重点）

#### 2.1.1 核心定位

整个多智能体系统的「调度中心、通信中枢、校验兜底中心」，不负责具体的内容创作或渲染工作，仅专注于「任务建模、动态调度、信息中转、结果校验、容错兜底」，是连接用户需求、各子智能体与前端的核心桥梁，决定架构的稳定性与高效性。

#### 2.1.2 核心功能（6大核心，无冗余）

1.  需求入口与分发：接收前端传递的用户自然语言需求（或修订需求），转发给需求解析智能体；后续接收各子智能体的输出结果，统一中转、整合。

2.  任务建模与调度：基于用户需求，用简化版DAG（有向无环图）拆解为结构化任务列表，定义任务依赖关系（如：框架设计未完成→无法启动资料研究；资料研究未完成→无法启动内容素材生成），按阶段顺序调度各子智能体执行，支持无依赖任务并行（如内容素材生成中可同步准备模板渲染的基础配置）。

3.  信息中转：作为所有子智能体的唯一通信节点，子智能体之间不直接交互，所有输入/输出均通过主智能体转发，确保信息传递的完整性与一致性，降低耦合。

4.  全流程校验：对每个子智能体的输出结果做「交叉校验」（子智能体自身已做自校验），重点校验「结果与需求的一致性、结果与框架的匹配性、各阶段结果的连贯性」（如：内容素材与研究结果是否匹配、渲染内容与模板是否适配），发现错误/冲突立即让对应智能体重跑，避免错误传递。

5.  容错兜底：为每个子智能体设置3次重试机制，某子智能体失效（如资料研究检索失败、渲染失败）时，自动降级为核心功能（如无研究资料则跳过研究环节，继续生成内容），确保整个PPT生成流程不中断、不卡死。

6.  进度与状态管理：记录每个子智能体的执行状态（待执行/执行中/完成/失败）、执行进度，整合为全局生成进度，反馈给前端用于实时展示；同时保存全流程所有中间结果，支持用户修订时仅重跑相关子智能体（如改数据仅重跑资料研究+内容素材，无需从头开始）。

#### 2.1.3 输入/输出（标准化JSON）

输入：

- 前端传递的用户需求（JSON）：含PPT主题、场景、行业、页数、模板类型、修订需求（可选）等；

- 各子智能体的输出结果（JSON）：含子智能体执行状态、自校验结果、核心数据（如需求结构化清单、PPT框架、研究结果等）。

输出：

- 向各子智能体下发的执行指令（JSON）：含任务ID、关联的中间结果、执行要求；

- 向前端反馈的全局进度（JSON）：含当前阶段、进度占比、执行状态、中间预览数据（可选）；

- 向PPT模板渲染智能体下发的最终整合数据（JSON）：含所有子智能体的有效输出，用于渲染生成PPT。

#### 2.1.4 关键设计（面试必讲）

采用「任务ID关联机制」，为每个用户需求生成唯一任务ID，所有子智能体的执行、中间结果、最终输出均绑定该任务ID，便于进度追溯、修订管理与错误排查；交叉校验采用「规则化配置」，可根据不同PPT场景（商务/校园/科技）调整校验规则，适配性强。

### 2.2 子智能体1：需求解析智能体（需求翻译官，不可替代）

#### 2.2.1 核心定位

将前端传递的「模糊用户自然语言需求」，转化为「所有子智能体可统一执行的结构化需求清单」，解决「单LLM需求解析不精准、遗漏核心要素、各子智能体执行标准不一致」的痛点，为后续所有环节定标、定规则。

#### 2.2.2 核心功能（4大核心，适配模板复用）

1.  核心要素提取：从用户自然语言中精准提取所有关键信息，无遗漏、无偏差，重点提取「模板相关要素」，适配PPT模板复用逻辑。

2.  模糊需求补全：对用户未明确的需求（如未指定模板类型、未明确页数），自动补全默认值（如默认商务风模板、默认15页），并标注补全说明，避免后续环节跑偏。

3.  需求结构化：将提取、补全后的需求，整理为标准化JSON格式的需求清单，明确各子智能体的执行标准（如模板类型、页数、风格、核心模块等）。

4.  自校验：对提取的要素做完整性、合理性校验（如页数不能为负数、模板类型需在预设模板库中），校验不通过则返回错误提示，由主智能体反馈给前端，让用户补充需求。

#### 2.2.3 提取的核心要素（全覆盖，无冗余）

- 基础信息：PPT主题、使用场景（商务汇报/校园答辩/产品宣讲）、行业、受众（领导/客户/学生）；

- 格式要求：页数、输出格式（pptx/pdf）、是否需要目录/页码/备注；

- 模板要求：模板类型（商务风/科技风/校园风，对应预设模板库中的模板）；

- 内容要求：核心模块（如前言/数据/问题/策略）、是否需要外部研究资料（关联资料研究智能体的开关）、是否上传本地数据（如Excel销售数据）；

- 特殊要求：用户自定义需求（如重点页标红、汇报人信息、字体要求）。

#### 2.2.4 输入/输出（标准化JSON示例）

输入：主协调智能体转发的用户自然语言需求（JSON），示例：{"user需求": "做一份2025电商618销售复盘PPT，商务风模板，15页，用于公司高层汇报，需要补充行业数据和竞品案例，上传了销售Excel数据，重点策略页标红"}

输出：结构化需求清单（JSON），示例：

  "ppt_topic": "2025电商618销售复盘",

### 2.3 子智能体2：框架设计智能体（PPT骨架师，不可替代）

#### 2.3.1 核心定位

根据需求解析智能体输出的「结构化需求清单」，设计符合「行业逻辑、受众需求、模板适配」的PPT整体框架，解决「单LLM生成PPT内容逻辑割裂、页序混乱、内容与模板布局不匹配」的痛点，为后续资料研究、内容创作定好「骨架」。

#### 2.3.2 核心功能（4大核心，适配模板复用）

1.  逻辑页序设计：按「行业场景+受众需求」设计PPT的页序（如电商复盘：前言→核心数据→行业对比→竞品分析→问题→策略→总结；校园答辩：研究背景→方法→实验→结论），确保逻辑连贯、重点前置。

2.  每页细节定义：为每一页设定「页码+标题+核心内容方向+是否需要图表/研究资料」，严格匹配用户要求的总页数，无多页/少页；同时标注模板适配要求（如数据页需匹配模板预留的图表区）。

3.  模板布局适配：结合预设模板的页面布局（如商务风模板的封面、目录、数据页、内容页版式），为每页标注「是否留图表位置」「是否留配图位置」，确保后续内容素材生成、渲染时，能精准适配模板，无需自定义排版。

4.  自校验：校验框架页数与用户需求是否一致、页序逻辑是否符合行业场景、核心模块是否全覆盖，校验不通过则返回主智能体，重新调整。

#### 2.3.3 设计原则（面试必讲）

1.  受众导向：给领导的PPT轻细节、重结果/问题/策略；给客户的PPT轻内部问题、重优势/价值；给学生的PPT重细节、重逻辑。

2.  行业适配：不同行业有固定的逻辑框架（如金融复盘重营收/利润/风险，教育宣讲重课程/效果/案例）。

3.  模板联动：框架设计严格贴合预设模板的版式，避免后续内容与模板布局冲突，减少渲染环节的调整成本。

### 2.4 子智能体3：资料研究智能体（内容支撑师，架构亮点）

#### 2.4.1 核心定位

本架构的核心亮点智能体，负责为PPT内容提供「外部权威资料支撑」，解决「原架构内容来源单一、缺乏行业数据/案例、说服力弱」的痛点，成为内容素材智能体的「数据/案例/素材库」，让PPT从「空壳内容」升级为「专业内容」。

核心逻辑：仅基于框架设计智能体标注的「需要研究」的页面，精准检索、筛选、整理资料，不做无意义的全网检索，提升研究效率；LLM仅负责信息筛选与结构化，不做虚构内容，确保资料的权威性与准确性。

#### 2.4.2 核心功能（3步走，极简高效）

1.  研究需求拆解：根据框架设计智能体输出的「PPT结构化框架表」，提取标注「is_need_research: true」的页面，拆解出具体的研究需求（如：第5页需研究「2025全网电商618 GMV、增长率、直播电商占比」；第6页需研究「同赛道TOP3竞品618销量、营销玩法」），明确研究范围与核心目标。

2.  多渠道精准检索：基于拆解的研究需求，从权威、高相关性的渠道检索/采集公开、可商用的资料，拒绝低质信息，核心检索渠道分为两类：

- 工程化采集：权威数据平台（艾瑞咨询、易观分析）、行业协会、政府官网、上市公司财报（通过爬虫/API调用采集公开数据）；

- LLM辅助检索：行业媒体（36氪、虎嗅）、竞品官网、社交媒体（小红书/抖音）、权威百科（维基百科、百度百科），通过LLM联网检索并筛选有效信息。

3.  资料结构化整理：将采集到的原始资料，通过LLM做「去重→筛选→提炼→结构化」处理，整理为与PPT框架严格匹配的研究结果包，标注资料来源（提升权威性），确保内容素材智能体可直接调用、无需二次处理；同时做自校验，确保资料与页面主题高度相关、数据准确。

#### 2.4.3 研究范围（精准匹配，不做无效研究）

完全由PPT框架决定，仅研究「框架中标注需要研究」的页面，研究内容分为3类：

- 数据类：行业整体数据、市场增长率、竞品数据、政策相关数据等（可可视化，用于图表生成）；

- 案例类：竞品案例、行业优秀案例、用户案例等（用于内容佐证，提升说服力）；

- 观点类：行业趋势预判、权威专家观点、政策解读等（用于内容升华，提升PPT深度）。

## 三、智能体协作流程（面试必讲，闭环逻辑）

### 3.1 核心协作逻辑（线性+并行结合）

用户需求 → 前端提交 → 主协调智能体 → 阶段调度子智能体（按顺序执行）→ 各子智能体自校验 → 主智能体交叉校验 → 渲染生成PPT → 前端实时预览 → 用户导出/修订

具体阶段流程：

1.  初始化阶段：用户在前端填写需求（含模板类型、是否需要研究资料等），提交后前端发送需求给主协调智能体，主智能体生成唯一任务ID，初始化任务状态为「待执行」。

2.  阶段1：需求解析（15%进度）→ 主智能体调度需求解析智能体，将用户需求转化为结构化需求清单，校验通过后，转发给框架设计智能体；前端实时展示「需求解析中→已完成」。

3.  阶段2：框架设计（30%进度）→ 主智能体调度框架设计智能体，基于结构化需求，生成PPT框架（含研究需求标注），校验通过后，转发给资料研究智能体；前端展示「框架设计中→已完成」。

4.  阶段3：资料研究（50%进度）→ 主智能体调度资料研究智能体，基于框架中的研究需求，检索、整理研究资料，生成研究结果包，校验通过后，转发给内容素材智能体；前端展示「资料研究中→已完成」（无研究资料则显示「未检索到研究资料，继续生成内容」）。

5.  阶段4：内容素材（80%进度）→ 主智能体调度内容素材智能体，融合研究结果、本地数据、框架要求，生成文字+素材，校验通过后，转发给PPT模板渲染智能体；前端开始接收预览数据，实时渲染PPT预览页面，用户可翻页查看。

6.  阶段5：模板渲染（100%进度）→ 主智能体调度PPT模板渲染智能体，加载模板、填充内容、生成可编辑PPT，校验通过后，返回文件路径与预览数据给主智能体；前端展示「渲染中→已完成」，显示导出按钮。

7.  后续阶段：用户点击导出按钮，前端调用后端接口，下载PPT文件；若用户提出修订需求，前端提交修订需求给主智能体，主智能体拆解修订点，仅调度相关子智能体重跑（如改模板仅重跑渲染智能体），重跑完成后，前端更新预览，用户可重新导出。

### 3.2 修订协作流程（面试亮点，体现用户体验）

用户修订需求 → 前端提交（任务ID+修订内容）→ 主协调智能体拆解修订点 → 调度对应子智能体重跑 → 主智能体校验新结果 → 调度渲染智能体重新渲染对应页面 → 前端更新预览 → 用户重新导出

示例：用户要求「补充更多竞品数据」→ 主智能体拆解修订点为「重新执行资料研究智能体（补充竞品数据）+ 内容素材智能体（更新文字/图表）+ 渲染智能体（更新对应页面）」→ 无需重跑需求解析、框架设计，提升修订效率。

## 四、设计亮点总结（面试主动讲，拉分关键）

1.  架构极简且落地性强：「1主5子」架构，砍掉冗余排版智能体，基于PPT模板复用，工程化实现渲染，无复杂逻辑，可通过Python+Autogen/LangChain+python-pptx快速落地demo，适配小型项目与面试展示。

2.  内容专业度升级：新增资料研究智能体，为PPT提供行业数据、案例、权威来源支撑，摆脱单纯的内容拼接，让PPT更具说服力，同时支持研究功能开关，适配不同场景。
