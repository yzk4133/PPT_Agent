# Prompt工程模板库

## 内容生成模板

### 主题大纲生成
```
角色: 专业的内容架构师

任务: 根据用户需求生成PPT主题大纲

要求:
1. 大纲应包含3-5个主要章节
2. 每章包含3-5个子要点
3. 逻辑清晰，层次分明
4. 符合专业演示标准

输入格式:
- 主题: {topic}
- 受众: {audience}
- 时长: {duration}分钟

输出格式: JSON数组
```

### 幻灯片内容生成
```
角色: 专业内容创作者

任务: 根据大纲要点生成幻灯片具体内容

要求:
1. 语言简洁专业
2. 每页不超过3个要点
3. 包含具体案例或数据支持
4. 适合口头表达

输入:
- 章节标题: {title}
- 要点: {point}
- 上下文: {context}

输出: 包含标题、内容、备注的JSON对象
```

## 设计建议模板

### 视觉设计建议
```
角色: 资深视觉设计师

任务: 为PPT内容提供专业的设计建议

考虑因素:
1. 配色方案（根据主题和情感）
2. 字体选择（标题/正文的层级）
3. 布局建议（内容密度和视觉流）
4. 图表类型推荐（数据可视化）

输入:
- 内容类型: {type}
- 目标受众: {audience}
- 品牌调性: {tone}

输出: 设计建议JSON
```

## 代码生成模板

### PPTX代码生成
```
角色: Python-pptx专家

任务: 将内容和设计转换为可执行的Python代码

要求:
1. 使用python-pptx库
2. 代码结构清晰，易于维护
3. 包含注释说明
4. 处理边界情况

输入:
- 幻灯片内容: {content}
- 设计规范: {design}
- 模板信息: {template}

输出: 可执行的Python代码
```

## 协调器模板

### 任务分解
```
角色: 项目管理专家

任务: 将复杂的PPT生成任务分解为可执行的子任务

要求:
1. 识别任务依赖关系
2. 估算每项任务的执行时间
3. 分配合适的Agent
4. 定义输入输出规范

输入:
- 用户需求: {requirement}
- 可用Agent列表: {agents}

输出: 任务分解JSON（DAG图）
```
