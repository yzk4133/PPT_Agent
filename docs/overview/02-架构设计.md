# 系统架构设计

## 🏗️ 整体架构

MultiAgentPPT 采用多智能体协作架构，通过 A2A（Agent-to-Agent）协议和 MCP（Model Context Protocol）实现智能体间的通信和能力扩展。

---

## 🔄 系统流程

```
用户需求
    ↓
┌─────────────────────────────────────────┐
│  1. 需求 → 大纲阶段                      │
│  • RAG 节点 + 检索 Agent                 │
│  • 生成结构化大纲                        │
│  • 实时反馈                              │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  2. 主题拆分                             │
│  • Topic-拆分 Agent                      │
│  • 划分子主题                            │
│  • 明确职责分配                          │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  3. 并行内容生成                         │
│  • 多个 Research Agent                   │
│  • MCP 获取外部数据                      │
│  • 并行抓取、分析、生成                   │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  4. 汇总与流式输出                       │
│  • Summary Agent 合并内容                │
│  • 流式并发推送前端                      │
│  • 实时展示进度                          │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  5. PPT 渲染与导出                       │
│  • 图文融合                              │
│  • 主题样式应用                          │
│  • 多格式导出（HTML/PDF/PPTX）            │
└─────────────────────────────────────────┘
```

## 📊 数据流设计

### Metadata 流转

```
前端传入 metadata
    ↓
{
  "language": "中文/English",
  "style": "formal/casual",
  "page_count": 10,
  "target_audience": "技术人员/普通用户"
}
    ↓
每个 Agent 获取并调整输出
    ↓
更新 metadata（中间状态 + 阶段性内容）
    ↓
返回前端展示
```

### 元数据作用

- **语言控制**：统一输出语言
- **风格调整**：内容语气和风格
- **进度跟踪**：每个 Agent 的执行状态
- **个性化**：根据目标受众调整内容

## 🎨 前端架构

### 渲染流程

```
Main.tsx
  → PresentationSlidesView.tsx (遍历 Slides)
    → SlideContainer.tsx (单张 Slide 容器)
      → presentation-editor.tsx (富文本编辑框架)
        ├── 标题渲染
```

- **Zustand**：全局状态管理（DetailLogs, 幻灯片内容）

## 🔄 架构演进

### 新架构 vs 旧架构

| 方面 | 旧架构 (slide_agent) | 新架构 (flat_slide_agent) | 改进 |
|------|---------------------|--------------------------|------|
| **结构** | 4层嵌套 | 3阶段扁平化 | 简化调试 |
| **性能** | 串行处理 | 并行研究 | 2.5x 提升 |
| **配置** | 硬编码 | 统一配置中心 | 易于管理 |
| **降级** | 基础重试 | 多层降级策略 | 更高可靠性 |
| **工具** | 直接导入 | Skill 框架 | 动态加载 |
| **记忆** | 无 | 持久化记忆 | 个性化体验 |

### 推荐使用

- **新项目**：`flat_slide_agent` + `flat_slide_outline`
- **高性能场景**：`flat_slide_agent`（并行研究）
- **向后兼容**：`slide_agent` + `slide_outline`
- **简单场景**：`simpleOutline` + `simplePPT`

## 🖼️ 多模态处理

### 图片处理流程

```
图片理解（Research Agent）
    ↓
抓取相关文章/搜索引擎图片
    ↓
图片生成（Flux, Midjourney）
    ↓
图片与内容融合
    ↓
PPT 页面布局
```

### 图片展示策略

- **背景图片**：`object-cover`（覆盖整个区域）
- **内容图片**：`object-contain`（完整显示）
- **加载失败**：自动隐藏（防止模型幻觉）

## 📄 PPT 页面结构

### 唯一性标识

```xml
<SECTION layout="left|right|vertical" page_number="1">
  <!-- 幻灯片内容 -->
</SECTION>
```

- **page_number**：作为唯一 key，支持覆盖更新
- **layout**：控制页面布局方式

### 输出格式

- **XML**：主要内容格式
- **JSON**：用于数据交换
- **Markdown**：表格和图表定义

## 🔧 技术协议

### A2A（Agent-to-Agent）

- **用途**：智能体间通信与任务协调
- **功能**：能力发现、任务分配、结果传递
- **提供商**：Google

### MCP（Model Context Protocol）

- **用途**：标准化接口
- **功能**：调用 API、数据库等外部工具
- **提供商**：Anthropic

### ADK（Agent Development Kit）

- **用途**：智能体开发工具包
- **功能**：整合 A2A 和 MCP
- **提供商**：Google
