## 🎯 核心概念

### 什么是MCP？

**MCP (Model Context Protocol)** = AI助手与外部工具通信的统一协议

- **统一消息格式**：JSON-RPC
- **统一接口规范**：tools/list, tools/call
- **传输层可选**：HTTP、stdio、WebSocket等

## 🔄 数据流向

### 智谱搜索调用流程

```
Agent.zhipu_search.invoke(query="AI")
        │
        ▼
Tool Registry 获取工具
        │
        ▼
ZhipuSearchTool.invoke()
        │
        ▼
HTTPMCPClient.call_tool("web_search_prime", query="AI")
        │
        ▼
HTTP POST 请求 → 智谱云端API
        │
        ▼
返回JSON-RPC响应
        │
        ▼
格式化结果 → 返回给Agent
```

### Zai视觉调用流程

```
Agent.zai_vision.invoke(image_url="xxx")
        │
        ▼
Tool Registry 获取工具
        │
        ▼
ZaiVisionTool.invoke()
        │
        ▼
StdioMCPClient.call_tool("vision_understand", image_url="xxx")
        │
        ▼
写入 stdin → npx进程 → 调用智谱API
        │
        ▼
读取 stdout ← JSON-RPC响应
        │
        ▼
格式化结果 → 返回给Agent
```

## 📦 组件职责

### 1. MCP客户端层

**职责**：处理底层通信，封装MCP协议细节

| 组件 | 职责 |
|------|------|
| **BaseMCPClient** | 定义统一接口（连接、断开、调用工具、列出工具） |
| **HTTPMCPClient** | 使用aiohttp发送HTTP请求，处理认证、超时 |
| **StdioMCPClient** | 启动子进程，管理stdin/stdout通信 |
| **MCPServerManager** | 加载配置，管理客户端生命周期 |

### 2. 工具包装层

**职责**：将MCP客户端包装成LangChain工具

| 组件 | 职责 |
|------|------|
| **ZhipuSearchTool** | 定义搜索参数模型，调用HTTP客户端，格式化搜索结果 |
| **ZaiVisionTool** | 定义视觉参数模型，调用stdio客户端，格式化分析结果 |

### 3. 工具注册层

**职责**：统一管理所有工具，提供给Agent使用

| 组件 | 职责 |
|------|------|
| **ToolRegistry** | 初始化MCP服务器，注册工具，提供工具查询接口 |

## 🔌 两种传输方式对比

| 维度 | HTTP (智谱搜索) | stdio (Zai视觉) |
|------|-----------------|-----------------|
| **通信方式** | HTTP POST请求 | stdin/stdout |
| **协议库** | aiohttp | asyncio subprocess |
| **连接管理** | 创建HTTP会话 | 启动/停止子进程 |
| **数据格式** | JSON-RPC over HTTP | JSON-RPC over stdin/stdout |
| **适用场景** | 远程云API | 本地CLI工具 |
| **优势** | 简单直接，跨语言 | 轻量，无网络开销 |

## 🔐 配置管理

### 配置文件结构

```yaml
# config/mcp_config.yaml
servers:
  zhipu-search:
    type: http              # 传输类型
    url: https://...         # API地址
    headers:                 # 认证信息
      Authorization: Bearer ${ZHIPU_API_KEY}

  zai-vision:
    type: stdio             # 传输类型
    command: npx            # 启动命令
    args: ["-y", "@z_ai/mcp-server"]
    env:                    # 环境变量
      Z_AI_API_KEY: ${ZAI_API_KEY}
```

### 环境变量

```bash
# .env
ZHIPU_API_KEY=xxx    # 智谱API密钥
ZAI_API_KEY=xxx      # Zai API密钥
```

## 🎨 关键设计决策

### 1. 为什么MCP客户端要独立成层？

**好处**：
- 职责分离：通信逻辑与业务逻辑解耦
- 可复用：其他项目也可以用这个客户端
- 易测试：可以mock客户端进行单元测试

### 2. 为什么需要MCPServerManager？

**好处**：
- 统一管理：一个地方管理所有MCP服务器
- 生命周期：统一处理启动和关闭
- 配置加载：自动替换环境变量

### 3. 为什么要包装成LangChain工具？

**好处**：
- 统一接口：Agent无需区分本地工具和MCP工具
- 参数验证：Pydantic自动验证输入参数
- 文档生成：工具描述自动暴露给LLM

## 📊 错误处理策略

### 三层错误处理

```
┌─────────────────────────────────────────┐
│         Agent层                          │
│  • 工具调用失败 → 降级处理或重试         │
└─────────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────────┐
│         工具层                           │
│  • 参数验证错误 → 返回友好提示            │
│  • 结果格式化错误 → 返回默认值           │
└─────────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────────┐
│         客户端层                         │
│  • 网络错误 → 自动重试                   │
│  • 协议错误 → 抛出明确异常               │
│  • 超时 → 取消请求                       │
└─────────────────────────────────────────┘
```

## 🚀 性能考虑

### 异步IO

- 所有MCP调用都是异步的
- 使用asyncio提高并发能力
- 支持同时调用多个工具

### 连接复用

- HTTP客户端复用TCP连接
- stdio进程保持常驻
- 避免重复创建连接开销

### 缓存策略

- 工具列表可以缓存
- 频繁调用的结果可以缓存（可选）
