## 🔧 5. 接口设计

### 5.1 公开接口

#### `async parse(user_input: str) -> PPTRequirement`

**功能**: 解析用户输入，返回结构化需求

**参数**:
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `user_input` | str | 是 | 用户自然语言输入 | "生成一份关于AI的PPT，15页" |

**返回值**:
| 类型 | 说明 |
|------|------|
| `PPTRequirement` | 结构化需求模型 |

**异常**:
| 异常类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| `ValueError` | 输入为空 | 抛出异常，提示用户 |
| `RequirementParseError` | LLM解析失败 | 自动降级，不抛出 |
| `ValidationError` | Pydantic验证失败 | 自动降级，不抛出 |

**使用示例**:
```python
agent = RequirementParserAgent()
requirement = await agent.parse("生成一份关于人工智能的PPT，15页")

print(requirement.ppt_topic)    # "人工智能"
print(requirement.page_num)     # 15
print(requirement.language)     # Language.ZH_CN
print(requirement.template_type) # TemplateType.BUSINESS
```

---

### 5.2 私有方法（内部接口）

#### `_parse_with_llm(user_input: str) -> PPTRequirement`

**功能**: 使用LLM解析（三层架构）

**流程**:
1. LLM生成JSON（带超时和重试）
2. JsonOutputParser解析
3. Pydantic验证 + 智能推导

**异常处理**:
- LLM超时 → 自动重试 → 仍失败则抛出异常
- JSON解析失败 → 自动重试 → 仍失败则抛出异常
- Pydantic验证失败 → 尝试修复 → 仍失败则抛出异常

---

#### `_parse_with_fallback(user_input: str) -> PPTRequirement`

**功能**: 规则降级解析

**保证**: 永远返回一个可用的 PPTRequirement

**提取规则**:
1. 主题：前100字符
2. 页数：正则匹配 `(\d+)\s*[_页页page]`
3. 语言：检测中文字符
4. 场景：关键词匹配（学术、创意、产品、培训）

---

#### `_validate_and_infer(raw_dict: dict) -> PPTRequirement`

**功能**: Pydantic验证 + 智能推导

**推导规则**:

| 已有字段 | 推导字段 | 推导规则 |
|----------|---------|---------|
| `scene=academic_presentation` | `template_type` | `academic_template` |
| `scene=product_launch` | `template_type` | `business_template` |
| `template_type=academic_template` | `tone` | `professional` |
| `template_type=creative_template` | `tone` | `creative` |
| `ppt_topic` | `core_modules` | 从主题中提取关键词 |

---

#### `_infer_defaults(raw_dict: dict) -> dict`

**功能**: 智能推导默认值

**默认值表**:
```python
DEFAULTS = {
    'page_num': 10,
    'language': 'ZH-CN',
    'template_type': 'business_template',
    'scene': 'business_report',
    'tone': 'professional',
    'core_modules': [],
    'need_research': False
}
```

---

#### `_fallback_parse(user_input: str) -> dict`

**功能**: 规则提取（LLM失败时使用）

**提取优先级**:
1. 显式指定（用户明确提到）
2. 关键词匹配
3. 默认值

**示例**:
```python
输入: "生成一份学术研究PPT，关于大语言模型，20页"
输出: {
    "ppt_topic": "学术研究PPT，关于大语言模型",
    "page_num": 20,
    "language": "ZH-CN",
    "template_type": "academic_template",
    "scene": "academic_presentation",
    "tone": "professional",
    "core_modules": []
}
```

---

#### `_format_validation_error(e: ValidationError) -> str`

**功能**: 将Pydantic错误转为用户友好的提示

**输入示例**:
```python
ValidationError(
    errors=[
        {'loc': ('page_num',), 'msg': 'ensure this value is greater than 5'},
        {'loc': ('ppt_topic',), 'msg': 'field required'}
    ]
)
```

**输出示例**:
```
• page_num的值太小
• 缺少必填字段：ppt_topic
```

## 🎨 6. Prompt设计详解

### 6.1 Few-Shot示例设计原则

| 示例类型 | 覆盖场景 | 包含字段 | 目的 |
|----------|---------|---------|------|
| 简单中文 | 基础商务 | 基础字段 | 理解基本格式 |
| 学术场景 | 学术模板 | 推导字段 | 展示场景识别 |
| 英文输入 | 英文处理 | 语言检测 | 展示多语言支持 |
| 创意场景 | 创意模板 | 风格字段 | 展示语调推导 |

### 6.2 Prompt优化技巧

#### 技巧1: 输出格式简化
```python
# ❌ 不推荐（Pydantic的完整schema）
"The output should be formatted as a JSON instance that conforms to the JSON schema below..."

# ✅ 推荐（简洁字段说明）
"请按照以下JSON格式输出（不要使用markdown包裹）：
{
    "ppt_topic": "PPT主题",
    "page_num": 页数（整数，5-50）,
    ...
}"
```

#### 技巧2: Few-Shot示例分布
```python
# 示例1: 简单场景（建立信心）
示例2: 复杂场景（展示能力）
示例3: 英文场景（展示多语言）
示例4: 边界场景（处理不完整输入）
```

#### 技巧3: 关键约束前置
```python
# ✅ 推荐：约束在开头
"请按照以下JSON格式输出（不要使用markdown包裹）..."

# ❌ 不推荐：约束在最后
"...请返回JSON格式，注意不要使用markdown包裹"
```

## 📐 7. 数据结构

### 7.1 PPTRequirement 模型

```python
{
    "ppt_topic": str,           # 必须，2-100字符
    "page_num": int,            # 必须，5-50
    "language": "ZH-CN" | "EN-US",  # 必须
    "template_type": "business_template" | "academic_template" | "creative_template",  # 推荐
    "scene": "business_report" | "academic_presentation" | "product_launch" | "training",  # 推荐
    "tone": "professional" | "casual" | "creative",  # 推荐
    "core_modules": [str, ...],  # 可选，最多10个
    "need_research": bool,       # 可选
    "color_scheme": str | None,  # 可选
    "target_audience": str | None  # 可选
}
```

### 7.2 智能推导映射表

```python
INFERENCE_RULES = {
    # scene → template_type
    "scene_to_template": {
        "academic_presentation": "academic_template",
        "product_launch": "business_template",
        "training": "business_template"
    },

    # template_type → tone
    "template_to_tone": {
        "academic_template": "professional",
        "creative_template": "creative",
        "business_template": "professional"
    },

    # 关键词 → scene
    "keywords_to_scene": {
        ("学术", "研究", "论文"): "academic_presentation",
        ("产品", "发布", "推介"): "product_launch",
        ("培训", "教程", "教学"): "training",
        ("报告", "总结", "汇报"): "business_report"
    }
}
```
