## 🎯 实施原则

### 核心原则
1. **TDD驱动**: 测试先行，先写测试用例定义期望行为
2. **自底向上**: 从最独立、无依赖的部分开始构建
3. **小步迭代**: 每次只改动一个文件，完成后立即验证
4. **逐步验证**: 每个Step完成后运行测试，确保通过

### 完成标准
- ✅ 所有测试用例通过
- ✅ 代码通过lint检查（pylint/flake8）
- ✅ 功能可演示（有简单示例）

## 🐛 常见问题

### Q1: BaseAgent不存在怎么办？

使用try-except处理导入失败，创建临时基类：
- 尝试导入BaseAgent
- 失败时创建临时基类（不影响开发）
- 后续替换为真实BaseAgent

### Q2: 如何避免真实LLM调用？

使用unittest.mock.AsyncMock：
- Mock agent.chain.ainvoke方法
- 返回预设的JSON字符串
- 测试时不调用真实API

### Q3: 测试失败如何调试？

1. 使用 `pytest -v` 查看详细错误
2. 使用 `pytest --pdb` 进入调试模式
3. 检查日志输出
