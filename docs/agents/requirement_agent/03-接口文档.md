## 🏷️ 2. 数据模型

### 2.1 PPTRequirement

**完整路径**: `backend.agents.core.requirements.models.PPTRequirement`

**继承**: `pydantic.BaseModel`

**字段定义**:

| 字段名 | 类型 | 必填 | 默认值 | 约束 | 说明 |
|--------|------|------|--------|------|------|
| `ppt_topic` | `str` | ✅ | - | 2-100字符 | PPT主题 |
| `page_num` | `int` | ✅ | - | 5 ≤ x ≤ 50 | PPT页数 |
| `language` | `Language` | ✅ | - | ZH-CN 或 EN-US | 语言代码 |
| `template_type` | `TemplateType` | ❌ | `TemplateType.BUSINESS` | 枚举 | 模板类型 |
| `scene` | `Scene` | ❌ | `Scene.BUSINESS_REPORT` | 枚举 | 使用场景 |
| `tone` | `Tone` | ❌ | `Tone.PROFESSIONAL` | 枚举 | 语调风格 |
| `core_modules` | `List[str]` | ❌ | `[]` | 最多10个 | 核心模块 |
| `need_research` | `bool` | ❌ | `False` | - | 是否需要研究 |
| `color_scheme` | `Optional[str]` | ❌ | `None` | - | 色彩方案 |
| `target_audience` | `Optional[str]` | ❌ | `None` | - | 目标受众 |

**方法**:

| 方法名 | 返回类型 | 说明 |
|--------|---------|------|
| `model_dump()` | `dict` | 转为字典（枚举转为值） |
| `model_dump_json()` | `str` | 转为JSON字符串 |

**使用示例**:
```python
from backend.agents.core.requirements import PPTRequirement, Language, TemplateType

# 创建模型
requirement = PPTRequirement(
    ppt_topic="人工智能",
    page_num=15,
    language=Language.ZH_CN,
    template_type=TemplateType.BUSINESS
)

# 访问字段
print(requirement.ppt_topic)    # "人工智能"
print(requirement.page_num)     # 15
print(requirement.language)     # Language.ZH_CN

# 转为字典
data = requirement.model_dump()
# {'ppt_topic': '人工智能', 'page_num': 15, 'language': 'ZH-CN', ...}

# 转为JSON
json_str = requirement.model_dump_json()
# '{"ppt_topic":"人工智能","page_num":15,"language":"ZH-CN",...}'
```

## 🤖 3. Agent类

### 3.1 RequirementParserAgent

**完整路径**: `backend.agents.core.requirements.requirement_agent.RequirementParserAgent`

**继承**: `BaseAgent`

#### 初始化方法

```python
def __init__(
    self,
    model: Optional[ChatOpenAI] = None,
    temperature: float = 0.0,
    agent_name: str = "RequirementParserAgent",
    timeout: int = 10,
    max_retries: int = 1
)
```

**参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `model` | `ChatOpenAI \| None` | ❌ | `None` | LangChain LLM实例（None则使用配置文件） |
| `temperature` | `float` | ❌ | `0.0` | LLM温度参数（0.0保证稳定输出） |
| `agent_name` | `str` | ❌ | `"RequirementParserAgent"` | Agent名称（用于日志） |
| `timeout` | `int` | ❌ | `10` | LLM调用超时时间（秒） |
| `max_retries` | `int` | ❌ | `1` | 最大重试次数 |

**使用示例**:
```python
# 使用默认配置
agent = RequirementParserAgent()

# 自定义配置
from langchain_openai import ChatOpenAI
model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)
agent = RequirementParserAgent(
    model=model,
    timeout=15,
    max_retries=2
)
```

#### parse() - 主解析方法

```python
async def parse(self, user_input: str) -> PPTRequirement
```

**功能**: 解析用户输入，返回结构化需求

**参数**:

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `user_input` | `str` | ✅ | 用户自然语言输入 | "生成一份关于AI的PPT，15页" |

**返回值**:

| 类型 | 说明 |
|------|------|
| `PPTRequirement` | 结构化需求模型 |

**异常**:

| 异常类型 | 触发条件 | 说明 |
|----------|----------|------|
| `ValueError` | 输入为空或空白 | 抛出异常，提示用户 |
| `RequirementParseError` | LLM解析完全失败且降级也失败 | 理论上不应触发（降级保证） |

**使用示例**:
```python
import asyncio
from backend.agents.core.requirements import RequirementParserAgent

async def main():
    agent = RequirementParserAgent()

    # 简单输入
    result = await agent.parse("生成一份关于人工智能的PPT，15页")
    print(result.ppt_topic)    # "人工智能"
    print(result.page_num)     # 15
    print(result.language)     # Language.ZH_CN

    # 复杂输入
    result = await agent.parse(
        "帮我做一个学术研究PPT，关于大语言模型，"
        "包含背景、原理、应用，20页左右"
    )
    print(result.template_type)  # TemplateType.ACADEMIC
    print(result.core_modules)   # ["背景", "原理", "应用"]

    # 英文输入
    result = await agent.parse(
        "Create a business presentation about Q3 sales, 10 pages"
    )
    print(result.language)       # Language.EN_US

asyncio.run(main())
```

## ⚠️ 4. 异常

### 4.1 RequirementParseError

**完整路径**: `backend.agents.core.requirements.requirement_agent.RequirementParseError`

**继承**: `Exception`

**用途**: 表示需求解析失败（理论上不应触发，因为有降级策略）

**属性**:

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `message` | `str` | 错误消息 |
| `original_error` | `Exception` | 原始异常 |

**使用示例**:
```python
from backend.agents.core.requirements import RequirementParseError

try:
    result = await agent.parse(user_input)
except RequirementParseError as e:
    print(f"解析失败: {e.message}")
    print(f"原始错误: {e.original_error}")
```

## 🔌 5. 使用场景

### 5.1 场景1: 基础使用

```python
from backend.agents.core.requirements import RequirementParserAgent

agent = RequirementParserAgent()
result = await agent.parse("生成一份关于人工智能的PPT，15页")
```

### 5.2 场景2: 批量解析

```python
from backend.agents.core.requirements import RequirementParserAgent

agent = RequirementParserAgent()

inputs = [
    "生成一份关于AI的PPT，15页",
    "帮我做一个Q3销售报告",
    "Create a presentation about climate change, 20 pages"
]

tasks = [agent.parse(inp) for inp in inputs]
results = await asyncio.gather(*tasks)

for result in results:
    print(f"{result.ppt_topic}: {result.page_num}页")
```

### 5.3 场景3: 集成到工作流

```python
# backend/agents/coordinator/master_graph.py

from backend.agents.core.requirements import create_requirement_parser
from langgraph.graph import StateGraph

# 创建节点
async def requirement_node(state: dict) -> dict:
    agent = create_requirement_parser()
    requirement = await agent.parse(state["user_input"])

    # 存储到状态中
    state["structured_requirements"] = requirement.model_dump()
    return state

# 构建工作流
workflow = StateGraph(PPTGenerationState)
workflow.add_node("requirement_parser", requirement_node)
```

### 5.4 场景4: 自定义LLM

```python
from langchain_openai import ChatOpenAI
from backend.agents.core.requirements import RequirementParserAgent

# 使用自定义模型
model = ChatOpenAI(
    model="gpt-4o",
    temperature=0.0,
    api_key="your-api-key"
)

agent = RequirementParserAgent(model=model)
result = await agent.parse("生成一份关于AI的PPT")
```

### 5.5 场景5: 错误处理

```python
from backend.agents.core.requirements import (
    RequirementParserAgent,
    RequirementParseError,
    ValueError
)

agent = RequirementParserAgent()

try:
    result = await agent.parse(user_input)
except ValueError as e:
    # 输入为空
    print(f"输入错误: {e}")
except RequirementParseError as e:
    # 解析完全失败（理论上不应触发）
    print(f"解析失败: {e.message}")
    # 使用默认需求
    result = PPTRequirement(
        ppt_topic="默认主题",
        page_num=10,
        language=Language.ZH_CN
    )
```

## 📚 8. 常见问题

### Q1: 如何处理空输入？

```python
try:
    result = await agent.parse("")  # 抛出 ValueError
except ValueError as e:
    print(f"输入不能为空: {e}")
```

### Q2: 如何获取JSON输出？

```python
result = await agent.parse("生成一份关于AI的PPT")
json_str = result.model_dump_json()
print(json_str)
```

### Q3: 如何自定义超时时间？

```python
agent = RequirementParserAgent(timeout=15)  # 15秒超时
```

### Q4: 降级策略是否保证一定成功？

是的，降级策略使用规则提取，保证永远返回一个可用的 `PPTRequirement` 模型。

### Q5: 如何禁用重试？

```python
agent = RequirementParserAgent(max_retries=0)  # 禁用重试
```
