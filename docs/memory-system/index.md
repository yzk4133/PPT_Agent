## 快速开始

### 系统简介

MultiAgentPPT 记忆系统是一个三层架构的智能记忆管理服务，专为多智能体协作场景设计。它通过自动化的数据流转、智能缓存和生命周期管理，实现了高性能、低成本的持久化记忆能力。

### 核心特性

- **三层记忆架构**: L1瞬时内存 → L2短期内存 → L3长期内存
- **自动提升机制**: 基于访问频率和重要性的智能数据流转
- **分布式锁**: Redis实现的并发控制
- **向量缓存**: 减少重复的API调用
- **生命周期管理**: 数据衰减、归档和自动清理

### 技术栈

| 组件 | 技术 |
|------|------|
| L1存储 | Python OrderedDict (内存) |
| L2存储 | Redis |
| L3存储 | PostgreSQL + pgvector |
| API框架 | FastAPI |
| 向量模型 | OpenAI text-embedding-3-small |

## 性能指标

### 当前系统性能

| 指标 | 数值 | 说明 |
|------|------|------|
| L1访问延迟 | <1ms | 纯内存操作 |
| L2访问延迟 | 1-5ms | Redis网络IO |

## 常见问题

### Q: 如何选择数据的初始存储层？

**A**: 系统会根据以下规则自动选择：

1. **重要性 ≥ 0.8** → L2短期内存
2. **用户级作用域** → L2短期内存
3. **默认** → L1瞬时内存

你也可以在写入时通过 `target_layer` 参数手动指定。

### Q: 数据什么时候会被提升到更高层？

**A**: 系统每5分钟自动扫描一次，满足以下条件会提升：

- **L1 → L2**: 访问≥3次 或 重要性≥0.7
- **L2 → L3**: 跨会话使用≥2次

### Q: 如何监控记忆系统的运行状态？

**A**: 通过REST API或直接调用：

```python
# 获取统计信息
stats = await memory_manager.get_stats()

# 查询API
curl http://localhost:8001/api/memory/stats
```

### Q: 过期的数据会自动清理吗？

**A**: 是的，系统有多层清理机制：

1. L1: 每60秒清理TTL过期数据
2. L2: Redis自动过期清理
3. L3: 生命周期管理器定期归档和清理冷数据
