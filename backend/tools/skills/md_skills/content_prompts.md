# 内容生成 Skill

> 根据研究资料生成高质量的幻灯片内容

---

## 📋 Level 1: 快速指南

### 核心步骤（5 步）

1. **分析需求** → 理解页面目标和受众
2. **提取要点** → 从资料中提取 3-5 个关键信息
3. **结构化内容** → 组织成标题、要点、细节
4. **优化表达** → 提升语言质量和吸引力
5. **质量检查** → 验证准确性和完整性

### 使用场景
- ✅ 内容页幻灯片生成
- ✅ 章节封面页内容
- ✅ 总结页内容整理

### 快速示例

**输入**：
```
主题: 深度学习在图像识别中的应用
受众: 大众
类型: 内容页
```

**输出**：
```
标题: AI 如何"看"图片？
要点:
- 像人脑一样学习特征
- 从数据中自动提取模式
- 准确率超越人类
```

## 📖 Level 2: 详细指南

### 步骤 1: 分析需求

理解页面的基本要求和约束条件。

**输入参数**：
| 参数 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `topic` | str | 页面主题 | - |
| `audience` | str | 目标受众 | expert / general / student |
| `page_type` | str | 页面类型 | content / cover / summary |

**输出格式**：
```json
{
  "objective": "页面的核心目标",
  "audience_level": "expert | general | student",
  "tone": "专业 | 轻松 | 严谨",
  "key_points_count": 3-5,
  "needs_subtitle": true/false
}
```

**受众适配策略**：
- **专家** (expert): 专业术语深入，高度详细
  - 示例: "基于 CNN 架构的深度学习模型在图像识别任务中..."
- **大众** (general): 简洁易懂，使用类比
  - 示例: "AI 像人脑一样，通过层层学习来识别图片..."
- **学生** (student): 循序渐进，增加示例
  - 示例: "想象一下，AI 就像一个学生，通过看很多图片来学习..."

---

### 步骤 2: 提取要点

从研究资料中提取关键信息点。

**输入**：
- 主题: `topic`
- 研究资料: `research_data`（来自 ResearchAgent）
- 要点数量: `num_points`（默认 5）

**提取原则**：
1. ✅ 每个要点一句话（10-20字）
2. ✅ 保持平行结构（都是动词开头，或都是名词开头）
3. ✅ 按重要性排序
4. ✅ 适合目标受众理解

**示例**：

*原始研究资料*：
```
深度学习是一种基于神经网络的机器学习方法。它通过多层网络结构
自动学习数据的层次化特征表示。在图像识别领域，卷积神经网络(CNN)
能够自动提取从边缘到形状再到完整物体的特征，准确率已超越人类。
```

*提取的要点*：
```
1. 像人脑一样学习特征
2. 从数据中自动提取模式
3. 准确率超越人类
```

**LLM 提示词模板**：
```
从以下研究资料中提取 {num_points} 个关键要点：

主题：{topic}
受众：{audience}

研究资料：
{research_data}

要求：
1. 每个要点一句话（10-20字）
2. 保持平行结构
3. 按重要性排序
4. 适合 {audience} 理解

返回 JSON 数组：["要点1", "要点2", ...]
```

### 步骤 3: 结构化内容

将要点组织成完整的内容结构。

**内容结构模板**：

```json
{
  "title": "页面标题（5-15字）",
  "subtitle": "副标题（可选，10-25字）",
  "key_points": [
    "要点1（10-20字）",
    "要点2（10-20字）",
    "要点3（10-20字）"
  ],
  "supporting_details": [
    {
      "point": "要点1",
      "detail": "支撑细节（20-30字）"
    }
  ],
  "notes": "演讲者备注（可选）"
}
```

**标题生成技巧**：

| 类型 | 技巧 | 示例 |
|------|------|------|
| **疑问式** | 用问句引发好奇 | "AI 如何'看'图片？" |
| **数字式** | 用数字突出重点 | "3 个步骤掌握..." |
| **对比式** | 用对比制造冲击 | "从 0 到 100：AI 的进化" |
| **隐喻式** | 用类比降低门槛 | "AI 的'眼睛'：卷积神经网络" |

**副标题生成技巧**：
- 补充说明主标题
- 提供上下文信息
- 简洁（10-25字）

**示例**：
- 主标题: "AI 如何'看'图片？"
- 副标题: "深度学习让机器拥有视觉"

### 步骤 4: 优化表达

提升内容的吸引力、清晰度和准确性。

**优化维度**：

| 维度 | 检查点 | 优化方法 | 示例 |
|------|--------|----------|------|
| **清晰度** | 是否容易理解？ | 简化长句、避免术语 | "基于卷积神经网络的深度学习模型" → "AI 通过层层学习识别图片" |
| **吸引力** | 是否引起兴趣？ | 使用问句/数字/对比 | "图像识别技术" → "AI 如何'看'图片？" |
| **准确性** | 是否准确表达？ | 核对原始资料 | 确保数据和引用正确 |
| **完整性** | 是否遗漏要点？ | 对照研究资料 | 检查是否覆盖核心内容 |
| **简洁性** | 是否精炼？ | 删除冗余词汇 | "在这个基础上" → "基于此" |

**优化提示词模板**：

```
优化以下内容，提升 {dimension}：

原文：
{original_content}

优化要求：
1. {specific_instruction}
2. 保持原意不变
3. 适合 {audience} 受众

返回优化后的内容。
```

**迭代优化策略**：

1. **第一轮**: 修复结构性问题
   - 标题长度（5-20字）
   - 要点数量（3-5个）
   - 结构完整性

2. **第二轮**: 优化表达
   - 要点长度（<50字）
   - 清晰度和吸引力
   - 语言流畅性

3. **第三轮**: 润色
   - 选词精准度
   - 语气一致性
   - 整体协调性

### 步骤 5: 质量检查

验证内容是否达到质量标准。

**检查清单**：

```python
def quality_check(content: Dict, requirement: Dict) -> Dict:
    """质量检查"""

    checks = {
        # 基本结构
        "has_title": bool(content.get("title")),
        "has_key_points": len(content.get("key_points", [])) >= 3,

        # 标题质量
        "title_length": 5 <= len(content.get("title", "")) <= 20,
        "title_attractive": not content.get("title", "").startswith("关于"),

        # 要点质量
        "key_points_count": len(content.get("key_points", [])) >= 3,
        "points_parallel": True,  # 检查是否平行结构
        "points_length_ok": all(len(p) <= 50 for p in content.get("key_points", [])),

        # 整体质量
        "has_evidence": len(content.get("supporting_details", [])) > 0,
        "audience_match": True  # 检查是否适合目标受众
    }

    score = sum(checks.values()) / len(checks)

    return {
        "score": score,
        "checks": checks,
        "passed": score >= 0.8
    }
```

**质量标准**：

| 项目 | 标准 | 权重 |
|------|------|------|
| 标题长度 | 5-20 字 | 15% |
| 标题吸引力 | 不使用套话 | 10% |
| 要点数量 | 3-5 个 | 20% |
| 要点长度 | < 50 字 | 15% |
| 平行结构 | 保持一致 | 15% |
| 支撑证据 | 有细节说明 | 15% |
| 整体分数 | ≥ 0.8 | - |

**质量分数含义**：
- **0.9-1.0**: 优秀，可以直接使用
- **0.8-0.9**: 良好，小幅优化即可
- **0.7-0.8**: 一般，需要优化
- **< 0.7**: 较差，需要重新生成

## 🔬 Level 3: 高级技巧

### 技巧 1: 上下文感知

考虑前后页面的上下文，生成连贯的内容。

**上下文因素**：
```
{
  "previous_topics": ["前一个主题", "前前一个主题"],
  "next_topics": ["后一个主题", "后后一个主题"],
  "position": 3,           # 当前页面位置
  "total": 10              # 总页面数
}
```

**上下文适配策略**：

| 位置 | 策略 | 内容特点 |
|------|------|----------|
| **开头** (position 1-2) | 引入主题 | 简洁、吸引兴趣、奠定基调 |
| **中间** (position 3-8) | 深入展开 | 详细、具体、逻辑连贯 |
| **结尾** (position 9-10) | 总结收尾 | 回顾、强调、留有余味 |

**实现示例**：

```python
def generate_with_context(
    self,
    page: Dict,
    previous_pages: List[Dict],
    next_pages: List[Dict]
) -> Dict:
    """考虑前后页面上下文"""

    position = len(previous_pages) + 1
    total = len(previous_pages) + 1 + len(next_pages)

    # 确定策略
    if position <= 2:
        strategy = "开头：简洁吸引"
    elif position >= total - 2:
        strategy = "结尾：总结强调"
    else:
        strategy = "中间：详细展开"

    # 生成内容
    prompt = f"""
    生成页面内容，考虑上下文：

    当前页面：{page['topic']}
    前面页面：{[p['topic'] for p in previous_pages]}
    后续页面：{[p['topic'] for p in next_pages]}
    位置：{position}/{total}

    策略：{strategy}

    要求：
    - 与前面内容保持连贯
    - 为后续内容做好铺垫
    - 根据位置调整深度
    """

    return self.llm.invoke(prompt)
```

### 技巧 2: 迭代优化

通过多轮迭代逐步提升内容质量。

**优化流程**：

```
初始内容
    ↓
质量检查 (分数 0.7)
    ↓
第一轮优化：修复结构 (分数 0.8)
    ↓
第二轮优化：改进表达 (分数 0.85)
    ↓
第三轮优化：润色细节 (分数 0.92)
    ↓
最终输出
```

**实现代码**：

```python
async def iterative_optimize(
    self,
    content: str,
    max_iterations: int = 3,
    target_score: float = 0.9
) -> str:
    """迭代优化内容"""

    for i in range(max_iterations):
        # 质量检查
        quality = self.quality_check(content)

        if quality["score"] >= target_score:
            logger.info(f"优化完成，第 {i+1} 轮，分数: {quality['score']}")
            break

        # 找出失败项
        failed_items = [
            name for name, passed in quality["checks"].items()
            if not passed
        ]

        logger.info(f"第 {i+1} 轮优化，修复: {failed_items}")

        # 优化提示词
        prompt = f"""
        优化以下内容，修复以下问题：{failed_items}

        内容：{content}

        要求：
        - 保持原意
        - 逐步提升质量
        """

        content = await self.llm.ainvoke(prompt)

    return content
```

**优化优先级**：

| 优先级 | 问题类型 | 优化方法 |
|--------|----------|----------|
| **P0** | 结构性问题 | 标题长度、要点数量、完整性 |
| **P1** | 表达问题 | 清晰度、吸引力、简洁性 |
| **P2** | 润色问题 | 选词、语气、流畅性 |

### B. 常见问题

**Q1: 如何选择要点数量？**
- 封面页: 1 个要点
- 内容页: 3-5 个要点
- 总结页: 3 个要点

**Q2: 标题多长合适？**
- 最短: 5 字（太短信息不足）
- 最长: 20 字（太长不易记忆）
- 推荐: 10-15 字

**Q3: 如何确保内容适合目标受众？**
- 专家: 使用专业术语，提供深度分析
- 大众: 使用类比，简化术语，强调应用
- 学生: 循序渐进，提供示例，给出学习路径

**Q4: 质量分数多少算合格？**
- ≥ 0.8: 合格，可以使用
- ≥ 0.9: 优秀，直接使用
- < 0.8: 需要优化或重新生成
