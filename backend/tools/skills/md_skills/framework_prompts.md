# 框架设计指南

## Level 1: 快速指南

### 核心原则

1. **封面**: 第1页，标题+副标题
2. **目录**: 第2页，展示整体结构
3. **内容**: 中间页面，核心内容展开
4. **总结**: 最后一页，回顾+致谢

### 标准框架结构

```
页码 | 类型 | 说明
-----|------|------
1    | cover | 封面
2    | directory | 目录
3-8  | content | 内容页
9    | summary | 总结
10   | thanks | 致谢
```

---

## Level 2: 详细指南

### 页面类型说明

| 类型 | 英文标识 | 用途 | 内容特点 |
|------|----------|------|----------|
| 封面 | cover | 开场 | 主标题、副标题、汇报人、日期 |
| 目录 | directory | 导航 | 章节列表、页码 |
| 内容页 | content | 核心信息 | 标题、要点（3-5个） |
| 图表页 | chart | 数据展示 | 图表类型、数据说明 |
| 图片页 | image | 视觉展示 | 图片描述、配文 |
| 总结 | summary | 回顾 | 核心要点总结 |
| 致谢 | thanks | 结束 | 感谢语、联系方式 |

### 框架设计流程

1. **分析需求**: 主题、页数、核心模块
2. **划分章节**: 将主题分解为3-5个章节
3. **分配页数**: 按重要性分配页数
4. **标记特殊页**: 图表、图片、研究需求

## Level 3: 高级技巧

### 技巧 1: 模块化设计

将PPT分解为可复用的模块：

```
主题：人工智能应用

模块1：概述（2页）
- 封面
- 目录

模块2：核心技术（3页）
- 技术原理
- 发展历程
- 关键算法

模块3：应用场景（3页）
- 医疗健康
- 智能交通
- 金融科技

模块4：总结（2页）
- 总结页
- 致谢页
```

### 技巧 2: 研究需求标记

为需要外部资料的页面标记：

```json
{
  "page_no": 5,
  "title": "AI在医疗领域的应用",
  "is_need_research": true,
  "keywords": ["AI医疗", "诊断准确率", "医疗案例"]
}
```

### 技巧 3: 图表分布策略

- 图表密度：每3-4页一个图表
- 类型多样化：柱状图、折线图、饼图
- 位置安排：重要论点后跟图表支撑
