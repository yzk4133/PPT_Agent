# 渐进式生成 vs 并行生成对比

## 场景：生成一个"人工智能发展史"的 PPT

---

## ❌ 当前方案：并行生成（Parallel Generation）

### 流程
```
用户输入：AI 发展史
    ↓
拆分成 10 个主题（并行）
    ↓
┌─────────┬─────────┬─────────┬─────────┐
│主题1:   │主题2:   │主题3:   │主题4:   │
│图灵测试 │专家系统 │机器学习 │深度学习│
└─────────┴─────────┴─────────┴─────────┘
    ↓         ↓         ↓         ↓
  独立调研   独立调研   独立调研   独立调研
    ↓         ↓         ↓         ↓
  独立生成   独立生成   独立生成   独立生成
```

### 问题示例

**第1页（主题1生成）:**
```markdown
# 图灵测试
- 1950年，艾伦·图灵提出
- 判断机器是否具有智能
- 很重要！！！
```

**第2页（主题2生成）:**
```markdown
# 专家系统
- 1970年代兴起
- MYCIN系统很成功
- 使用if-then规则
```

**第3页（主题3生成）:**
```markdown
# 深度学习
- 2012年AlexNet
- 神经网络
- （突然提到）图灵测试现在也能通过了
```

### ❌ 出现的问题
1. **风格不一致**：第1页用感叹号，第2页很正式
2. **逻辑割裂**：第3页提到图灵测试，但第1页已经讲过
3. **缺少过渡**：每页都是独立的，缺少"然而""随后"这样的连接词
4. **重复调研**：多个主题都调研了相似的内容

---

## ✅ 渐进式生成（Progressive Generation）

### 流程
```
用户输入：AI 发展史
    ↓
【全局规划阶段】（一次性）
    ↓
生成：
- 大纲：[图灵测试 → 专家系统 → 机器学习 → 深度学习]
- 风格指南："学术风格，使用时间线串联，每页不超过5个要点"
- 关键论点："每个时代都是对前一个时代的突破"
    ↓
【渐进式生成阶段】（顺序生成）

生成第1页 → 缓存到上下文 → 生成第2页 → 缓存到上下文 → ...
```

### ✅ 实际效果

**第1页生成：**

## 🎯 核心区别总结

| 维度 | 并行生成 | 渐进式生成 |
|------|----------|------------|
| **上下文** | 每页独立生成 | 每页都看到前面的内容 |
| **连贯性** | ❌ 容易割裂 | ✅ 像讲故事一样连贯 |
| **风格** | ❌ 可能不一致 | ✅ 全局统一 |
| **重复** | ❌ 可能重复调研 | ✅ 复用已生成内容 |
| **过渡** | ❌ 缺少连接 | ✅ 自然过渡 |
| **用户控制** | ❌ 等待全部生成完 | ✅ 实时看到，可随时调整 |
| **API调用** | 10页 = 10次调研 + 10次生成 | 10页 = 1次规划 + 10次生成 |

---

## 💡 通俗类比

### 并行生成 = 10个人各自写一章书
- 第1章：张三写（不知道后面写什么）
- 第2章：李四写（不知道第1章写了什么）
- ...
- 最后拼起来 → 发现风格不统一，逻辑不通顺

### 渐进式生成 = 1个人按顺序写书
- 先写大纲：全书脉络
- 写第1章 → 知道后面要写什么，埋下伏笔
- 写第2章 → 回顾第1章，承上启下
- 写第3章 → 总结前两章，引出下文
- ...
- 自然连贯，像讲故事

## 🚀 实现伪代码

### 当前并行方案
```python
async def parallel_generate(topic, num_slides=10):
    # 1. 拆分主题
    subtopics = await split_topic(topic, num_slides)

    # 2. 并行调研（10个API调用）
    research_tasks = [research_subtopic(t) for t in subtopics]
    research_results = await asyncio.gather(*research_tasks)

    # 3. 并行生成（10个API调用）
    slide_tasks = [
        generate_slide(subtopics[i], research_results[i])
        for i in range(num_slides)
    ]
    slides = await asyncio.gather(*slide_tasks)

    return slides  # 拼起来，可能不连贯
```

### 渐进式方案
```python
async def progressive_generate(topic, num_slides=10):
    # 1. 全局规划（1次API调用，使用大模型）
    plan = await global_planner(
        topic=topic,
        num_slides=num_slides,
        instruction="生成大纲、风格指南、关键论点、时间线"
    )
    # plan = {
    #     "outline": ["图灵测试", "专家系统", ...],
    #     "style": "学术风格，使用时间线",
    #     "narrative_arc": "从理论到实践到突破",
    #     "key_arguments": ["每个时代都是突破"]
    # }

    # 2. 渐进式生成（10次API调用，使用中小模型）
    slides = []
    context = {
        "plan": plan,
        "previous_slides": [],  # 动态增长
    }

    for i, slide_topic in enumerate(plan.outline):
        # 生成当前页（基于前面所有页）
        slide = await generate_slide(
            topic=slide_topic,
            context={
                **context,
                "transition_hint": f"这是第{i+1}页，前面有{i}页内容",
                "narrative_goal": plan.narrative_arc,
                "style_guide": plan.style
            }
        )

        slides.append(slide)

        # 更新上下文（下一页能看到这一页）
        context["previous_slides"].append({
            "title": slide.title,
            "key_points": slide.points
        })

        # 流式返回给用户
        yield slide

    return slides
```

## 📊 成本对比

### 场景：生成10页PPT

#### 并行生成（当前方案）
```
调研阶段：10个API调用 × GPT-4 = 10 × $0.10 = $1.00
生成阶段：10个API调用 × GPT-4 = 10 × $0.05 = $0.50
质量检查：3个API调用 × GPT-4 = 3 × $0.03 = $0.09
─────────────────────────────────────
总计：$1.59（23次API调用）
```

#### 渐进式生成（优化方案）
```
规划阶段：1次API调用 × GPT-4 = 1 × $0.15 = $0.15  （用大模型保证质量）
生成阶段：10次API调用 × GPT-3.5 = 10 × $0.01 = $0.10  （中小模型）
检查阶段：1次整体审核 × GPT-4 = 1 × $0.10 = $0.10
─────────────────────────────────────
总计：$0.35（12次API调用）

💰 节省 78% 成本
⏱️ 减少 47% API调用
✅ 内容更连贯
```

## 🎯 总结

### 渐进式生成的本质
1. **全局规划，局部生成**
   - 先看到全局（大纲、风格、故事线）
   - 再逐个生成（每一步都知道自己在全局中的位置）

2. **上下文传递**
   - 后面的页面能看到前面的内容
   - 像人写文章一样连贯

3. **实时反馈**
   - 用户可以边看边调整
   - 不用等全部生成完

4. **成本优化**
   - 大模型做规划（1次）
   - 小模型做生成（N次）
   - 比全部用大模型便宜很多

### 适用场景
✅ 需要连贯性的内容（PPT、文章、教程）
✅ 用户需要实时预览
✅ 成本敏感的应用
❌ 真正可以并行的独立任务（如批量生成摘要）
