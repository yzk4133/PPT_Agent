## 🔢 常用计算公式

### LoRA 参数量计算

```python
# 单模块参数
单模块 = rank × (input_dim + output_dim)
       = 8 × (512 + 512)
       = 8,192

# 总参数（7 个模块 × 8 层）
总参数 = 8,192 × 7 × 8 ≈ 131K

# 占比
占比 = 131K / 26M ≈ 0.5%
```

### ROI 计算

```python
# 每天成本
大模型 = 100,000 × $0.02 = $2,000
小模型 = 100,000 × $0.001 = $100
节省 = $1,900/天

# 回本周期
回本 = $50 / $1,900 ≈ 0.026 天（如果已有模型）
回本 = $50 / ($2,000 - $100) ≈ 38 天（从大模型迁移）
```

### 置信度计算

```python
def compute_confidence(logits):
    probs = F.softmax(logits, dim=-1)
    max_prob = probs.max(dim=-1).values
    return max_prob.mean().item()

# 阈值
置信度 < 0.7 → Fallback 到大模型
```

## 🎯 可能的追问速答

| 追问 | 30 秒答案 |
|------|-----------|
| **LoRA 参数量怎么算？** | rank × (input_dim + output_dim) × 模块数 × 层数 = 8 × 1024 × 7 × 8 ≈ 131K |
| **为什么 rank=8？** | 经验值，hidden_size 的 1-2%，实验发现 rank=8 和 16 效果接近 |
| **数据量怎么定？** | LoRA 只训练 131K 参数，几百条就够，1000 条是性价比拐点 |
| **如何验证数据质量？** | 三层：JSON 格式验证 + 人工抽检 10% + 验证损失监控 |
| **置信度怎么算？** | 输出概率的 max probability，阈值 0.7，低于就 Fallback |
| **<100ms 怎么实现？** | 小模型本身快 + KV Cache + 查询缓存，实测 20-50ms |
| **如果数据扩大 10 倍？** | 可能调大 rank（8→16），注意数据质量，防止过拟合 |
| **误判怎么办？** | 置信度 < 0.7 Fallback 到大模型，误判率 8%，最终 95%+ |
