## 训练全景图

```
┌─────────────────────────────────────────────────────────────┐
│              MiniMind 完整训练流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                          │
│  数据准备 → 分词器训练 → 预训练 → SFT → 后续优化        │
│      │           │            │      │       │             │
│      ▼           ▼            ▼      ▼       ▼             │
│  收集文本   BPE/Unigram   学语言  学对话  提升能力         │
│  清洗格式   构建6400词表  基础   格式   或适应领域        │
│                                    知识                   │
└─────────────────────────────────────────────────────────────┘
```

### 各阶段核心目标

| 阶段 | 输入 | 输出 | 核心目标 |
|-----|------|------|---------|
| **数据准备** | 原始文本 | 清洗后的 jsonl | 高质量训练数据 |
| **预训练** | 大规模文本 | `pretrain_*.pth` | 学习语言基础知识 |
| **SFT** | 对话数据 | `full_sft_*.pth` | 学习任务格式 |
| **LoRA** | 特定任务数据 | `lora_*.pth` | 适配特定领域 |
| **DPO** | 偏好数据 | `rlhf_*.pth` | 优化回复质量 |
| **蒸馏** | 大模型输出 | `distill_*.pth` | 迁移大模型能力 |

## 阶段一：预训练 (Pretrain)

### 目标

让模型"喝"饱知识，学习语言的统计规律和基础常识。

### 原理解释

```
预训练就像让婴儿大量听人说话：

输入: "秦始皇是中国历史上第一位皇帝"
      ↓
模型学习: "秦始皇" 后面经常接 "是"、"中国"、"皇帝"
      ↓
能力: 可以进行词语接龙，但不会对话

示例:
输入: "秦始皇"
输出: "是中国的第一位皇帝，统一六国，建立了秦朝"
```

### 训练步骤

#### 1. 数据准备

```bash
# 目录结构
dataset/
├── pretrain_hq.jsonl       # 1.6GB 高质量中文语料
├── pretrain_large.jsonl     # 大规模预训练数据
└── pretest_domain.jsonl     # 领域特定数据
```

**数据格式**：
```json
{"text": "如何才能摆脱拖延症？治愈拖延症并不容易，但以下建议可能有所帮助..."}
{"text": "光合作用是植物、藻类和某些细菌利用光能将二氧化碳和水转化为有机物的过程..."}
```

#### 2. 启动训练

```bash
cd trainer

python train_pretrain.py \
  --data_path ../dataset/pretrain_hq.jsonl \
  --epochs 1 \
  --batch_size 32 \
  --learning_rate 5e-4 \
  --max_seq_len 512 \
  --hidden_size 512 \
  --num_hidden_layers 8 \
  --save_interval 100
```

#### 3. 训练监控

```
预期输出：
Epoch:[1/1](100/5000) loss:2.453 lr:0.000500 epoch_Time:35min
Epoch:[1/1](200/5000) loss:2.123 lr:0.000499 epoch_Time:34min
Epoch:[1/1](300/5000) loss:1.987 lr:0.000498 epoch_Time:33min
...

Loss 下降趋势：
初始 loss: ~8-10
中期 loss: ~2-3
最终 loss: ~1.5-2.0
```

#### 4. 测试预训练模型

```bash
python eval_model.py --model_mode 0 --hidden_size 512

# 预训练模型只会接龙，不会对话
👶: 马克思主义基本原理
🤖: 是关于自然、社会和思维发展一般规律的科学
```

### 关键参数说明

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| `--epochs` | 1 | 训练轮数，1轮足够快速验证 |
| `--batch_size` | 32 | 批量大小，显存不足可减至16 |
| `--learning_rate` | 5e-4 | 学习率 |
| `--max_seq_len` | 512 | 最大序列长度 |
| `--hidden_size` | 512 | 隐藏层维度（影响模型大小） |
| `--num_hidden_layers` | 8 | Transformer层数 |
| `--accumulation_steps` | 8 | 梯度累积步数 |

## 阶段二：监督微调 (SFT)

### 目标

让已经"喝饱"知识的模型学会对话格式和任务规范。

### 原理解释

```
SFT 就像教大学生说话礼仪：

预训练模型（有知识但无格式）：
用户: "你好"
模型: "你好你好大家好好你好好..."  ❌ 只会重复

SFT模型（学会对话格式）：
用户: "你好"
模型: "你好！有什么我可以帮助你的吗？"  ✅ 符合格式

数据格式：
{
  "conversations": [
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的？"}
  ]
}
```

### 训练步骤

#### 1. 数据准备

```bash
dataset/
├── sft_mini_512.jsonl     # 1.2GB 快速训练数据
├── sft_512.jsonl          # 7.5GB 标准训练数据
└── sft_2048.jsonl         # 9GB 长文本训练数据
```

**数据格式**：
```json
{
  "conversations": [
    {"role": "user", "content": "请介绍一下自己。"},
    {"role": "assistant", "content": "我是人工智能助手，可以帮助您解答问题、提供信息。"}
  ]
}
```

#### 2. 启动训练

```bash
# 加载预训练权重，训练对话能力
python train_full_sft.py \
  --data_path ../dataset/sft_mini_512.jsonl \
  --epochs 1 \
  --batch_size 32 \
  --learning_rate 5e-4 \
  --max_seq_len 512 \
  --hidden_size 512 \
  --num_hidden_layers 8 \
  --save_interval 100
```

#### 3. 训练过程

```
加载预训练权重: ../out/pretrain_512.pth
开始SFT训练...

Epoch:[1/1](100/3000) loss:1.234 lr:0.000500
Epoch:[1/1](200/3000) loss:0.987 lr:0.000499
...

Loss 下降：
初始 loss: ~1.5-2.0
最终 loss: ~0.6-0.9
```

#### 4. 测试对话能力

```bash
python eval_model.py --model_mode 1 --hidden_size 512

👶: 你好
🤖: 你好！有什么我可以帮助你的吗？

👶: 请介绍一下自己
🤖: 我是人工智能助手，通过自然语言处理技术与您交流。
```

## 阶段三：优化训练 (可选)

根据需求选择以下一种或多种训练方式：

### 选项 A: LoRA 微调（推荐特定任务）

**适用场景**：意图识别、领域适配、格式化输出

```bash
python train_lora.py \
  --data_path ../dataset/lora_intent.jsonl \
  --base_model ../out/full_sft_512.pth \
  --lora_r 16 \
  --lora_alpha 32 \
  --epochs 3

# 输出: ../out/lora/lora_intent_512.pth
```

**优势**：
- ✅ 只训练 1-2% 参数
- ✅ 训练速度快（10-30分钟）
- ✅ 可随时切换/禁用
- ✅ 多个 LoRA 可叠加

### 选项 B: DPO 强化学习

**适用场景**：提升回复礼貌性、减少有害内容

```bash
python train_dpo.py \
  --data_path ../dataset/dpo.jsonl \
  --base_model ../out/full_sft_512.pth \
  --epochs 1

# 数据格式：包含 chosen 和 rejected 对
```

**数据格式**：
```json
{
  "chosen": [
    {"role": "user", "content": "如何赚钱？"},
    {"role": "assistant", "content": "可以通过工作、创业、投资等正当途径..."}
  ],
  "rejected": [
    {"role": "user", "content": "如何赚钱？"},
    {"role": "assistant", "content": "我无法提供相关建议。"}
  ]
}
```

### 选项 C: 知识蒸馏

**适用场景**：让小模型学习大模型能力

```bash
# 用 Qwen/GPT 等大模型生成训练数据
# 然后进行 SFT 训练
python train_full_sft.py \
  --data_path ../dataset/sft_distill.jsonl
```

### 选项 D: 推理能力训练

**适用场景**：学习 DeepSeek-R1 的思考模式

```bash
python train_distill_reason.py \
  --data_path ../dataset/r1_mix_1024.jsonl \
  --base_model ../out/full_sft_512.pth

# 数据格式：包含 <think> 标签的思考过程
```

## 训练检查点

### 预训练完成标准

- [ ] Loss 稳定在 1.5-2.0
- [ ] 模型能完成基本的词语接龙
- [ ] 没有出现 NaN loss
- [ ] 生成的文本语法基本正确

### SFT完成标准

- [ ] Loss 稳定在 0.6-0.9
- [ ] 能正确识别对话格式
- [ ] 回复内容相关性 >70%
- [ ] JSON格式输出基本正确

### 部署前检查

- [ ] 模型文件完整（.pth 或 gguf）
- [ ] 推理脚本运行正常
- [ ] 响应速度可接受（<3秒）
- [ ] 内存/显存占用合理

## 常见配置推荐

### 快速原型验证

```bash
# 最小数据集 + 最小模型
预训练: pretrain_hq.jsonl (1 epoch)
SFT: sft_mini_512.jsonl (1 epoch)
总时间: ~2小时
适用: 快速验证想法
```

### 标准训练

```bash
# 完整数据集 + 中等模型
预训练: pretrain_hq.jsonl (2 epochs)
SFT: sft_512.jsonl (2 epochs)
总时间: ~15小时
适用: 一般应用场景
```

### 高质量训练

```bash
# 大规模数据 + 大模型
预训练: pretrain_large.jsonl (3 epochs)
SFT: sft_2048.jsonl (2 epochs)
DPO: dpo.jsonl (1 epoch)
总时间: ~30-40小时
适用: 生产级部署
```
