## 配置参数说明

### 核心参数

```python
# 模型架构参数
MiniMindConfig(
    # ===== 基础配置 =====
    vocab_size=6400,           # 词表大小（固定）
    hidden_size=512,            # 隐藏层维度 ⭐ 最重要
    num_hidden_layers=8,        # Transformer层数 ⭐ 第二重要

    # ===== 注意力配置 =====
    num_attention_heads=8,       # 注意力头数
    num_key_value_heads=2,       # KV头数（GQA）

    # ===== RoPE配置 =====
    rope_theta=1000000.0,       # RoPE基频
    max_position_embeddings=32768,  # 最大位置

    # ===== MoE配置（可选）=====
    use_moe=False,               # 是否使用MoE
    n_routed_experts=4,          # 路由专家数
    n_shared_experts=1,          # 共享专家数

    # ===== 训练配置 =====
    dropout=0.0,                # Dropout率
    rms_norm_eps=1e-5,          # RMSNorm参数
)
```

### 参数影响分析

#### hidden_size (隐藏层维度)

```python
影响模型宽度，决定每层的信息容量

参数量影响: ∝ hidden_size²

配置建议:
- 256: 极小模型（~10M）- 教学演示
- 512: 小型模型（~26M）- 推荐入门 ✅
- 640: 中型模型（~50M）- 平衡选择
- 768: 标准模型（~104M）- 推荐进阶 ✅
- 1024: 大型模型（~180M）- 高性能
```

#### num_hidden_layers (层数)

```python
影响模型深度，决定抽象能力层级

参数量影响: ∝ num_hidden_layers

配置建议:
- 6:  快速训练（~20M）
- 8:  标准配置（~26M）- 推荐入门 ✅
- 12: 深度优先（~70M）
- 16: 高深度（~104M）- 推荐进阶 ✅
- 24: 超深度（~150M）- 更强抽象
```

#### num_attention_heads (注意力头数)

```python
影响注意力机制的数量

配置建议:
- 必须能被 hidden_size 整除
- 建议 hidden_size / num_attention_heads = 64 (头维度)

示例:
hidden_size=512  → num_attention_heads=8   (每头64维)
hidden_size=768  → num_attention_heads=12  (每头64维)
hidden_size=1024 → num_attention_heads=16  (每头64维)
```

#### num_key_value_heads (KV头数)

```python
GQA (分组查询注意力) 配置

影响:
- 推理时的 KV Cache 大小
- 训练速度
- 模型质量（轻微）

配置建议:
- 2:  高压缩比（推理快，质量略降）
- 4:  中等压缩（平衡）✅
- 8:  低压缩（质量优先）
- 等于 num_attention_heads: 标准MHA（无压缩）

公式:
- 压缩比 = num_attention_heads / num_key_value_heads
- 8头 / 2 KV头 = 4倍压缩
```

## 调优策略

### 宽度 vs 深度

```
同等参数量下的两种策略:

方案A: 宽而浅 (如: 1024×6)
├─ 优势: 并行计算能力强
├─ 劣势: 抽象层级有限
└─ 适合: 图像、时序数据

方案B: 窄而深 (如: 512×16) ✅推荐
├─ 优势: 抽象层级丰富
├─ 劣势: 训练稍慢
└─ 适合: 语言、推理任务

研究结论:
"深而窄"的架构在LLM任务中表现更好
```

### 训练效率优化

```python
# 1. 混合精度训练
scaler = torch.cuda.amp.GradScaler()
with torch.cuda.amp.autocast():
    # 训练代码
效果: 2x加速，显存减半

# 2. 梯度累积
batch_size = 16
accumulation_steps = 4
# 有效批量 = 16 × 4 = 64
效果: 更大batch，更稳定训练

# 3. Flash Attention
config.flash_attn = True
效果: 注意力计算加速30%

# 4. GQA (分组查询注意力)
num_key_value_heads = 2  # 而非8
效果: KV Cache减少75%
```

### 超参数调优

```python
# 学习率调整
小模型 (26M):   lr = 5e-4  # 标准
中模型 (104M):  lr = 3e-4  # 略低
大模型 (1B+):   lr = 1e-4  # 更低

# Batch size调整
显存充足: batch_size = 32 或更大
显存紧张: batch_size = 8, accumulation_steps = 4

# 序列长度
快速训练: max_seq_len = 256 或 512
标准训练: max_seq_len = 512
长文本:   max_seq_len = 1024 或 2048

# Warmup
total_steps = 10000
warmup_iters = 100  # 前10%步数线性warmup
```

## 配置检查清单

### 训练前检查

- [ ] hidden_size 能被 num_attention_heads 整除
- [ ] 显存足够（至少2×模型大小）
- [ ] 学习率根据模型大小调整
- [ ] batch_size 合理（显存利用率80-90%）
- [ ] max_seq_len 与数据匹配
- [ ] 混合精度训练已启用

### 性能基准

- [ ] 训练速度: >100 samples/s (3090)
- [ ] Loss收敛: 预训练<2.0, SFT<0.9
- [ ] 推理速度: >20 tokens/s (FP16)
- [ ] 显存占用: <80%

## 常见问题

### Q1: hidden_size 和 layers 哪个更重要？
A: **layers** (深度) 对语言模型更重要。研究显示"深而窄"优于"宽而浅"。

### Q2: 可以改变 vocab_size 吗？
A: 不建议。6400词表是精心设计的，改变需要重新训练分词器。

### Q3: MoE 一定比 Dense 好吗？
A: 不一定。MoE推理更慢，训练更复杂。只在参数预算有限但追求效果时使用。

### Q4: 如何选择 num_key_value_heads？
A: 推荐设为 num_attention_heads 的 1/4 或 1/2。压缩比越大，推理越快，但质量略降。
