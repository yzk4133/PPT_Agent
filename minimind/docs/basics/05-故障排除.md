## 环境配置问题

### ❌ 问题：CUDA 不可用

**现象**：
```python
torch.cuda.is_available()  # 返回 False
```

**解决方案**：

```bash
# 1. 检查 CUDA 版本
nvidia-smi

# 2. 安装匹配的 PyTorch
# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# CUDA 12.1
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# 3. 验证
python -c "import torch; print(torch.cuda.is_available())"
```

### ❌ 问题：依赖包冲突

**现象**：
```
ImportError: cannot import name 'amp' from 'torch.cuda'
ModuleNotFoundError: No module named 'transformers'
```

**解决方案**：

```bash
# 1. 卸载旧版本
pip uninstall torch torchvision torchaudio transformers

# 2. 清理缓存
pip cache purge

# 3. 重新安装
pip install -r requirements.txt

# 4. 验证安装
python -c "import transformers; print(transformers.__version__)"
python -c "import torch; print(torch.__version__)"
```

### ❌ 问题：显存不足

**现象**：
```
RuntimeError: CUDA out of memory. Tried to allocate 2.00 GiB
```

**解决方案**：

```python
# 方案1: 减小批量大小
parser.add_argument("--batch_size", type=int, default=8)  # 从32减到8

# 方案2: 减小序列长度
parser.add_argument("--max_seq_len", type=int, default=256)  # 从512减到256

# 方案3: 启用梯度检查
# 在模型配置中添加
config.gradient_checkpointing = True

# 方案4: 增加梯度累积
parser.add_argument("--accumulation_steps", type=int, default=8)  # 累积8步

# 方案5: 使用更小的模型
# MiniMind2-Small (26M) 而非 MiniMind2 (104M)
```

## 训练阶段问题

### ❌ 问题：Loss 为 NaN

**现象**：
```
Epoch:[1/1](100/5000) loss:nan lr:0.000500
```

**原因分析**：
- 学习率过高
- 梯度爆炸
- 数据异常值

**解决方案**：

```python
# 1. 降低学习率
parser.add_argument("--learning_rate", type=float, default=1e-4)  # 从5e-4降到1e-4

# 2. 增加梯度裁剪
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 3. 检查数据
def validate_data(file_path):
    with open(file_path, 'r', encoding='utf-8') as f:
        for line in f:
            data = json.loads(line)
            # 检查是否有异常值
            for conv in data['conversations']:
                if len(conv['content']) == 0:
                    print("发现空内容！")

# 4. 使用混合精度训练
scaler = torch.cuda.amp.GradScaler()
with torch.cuda.amp.autocast():
    # 训练代码
```

### ❌ 问题：Loss 不下降

**现象**：
```
Epoch:[1/1](100/5000) loss:2.456 lr:0.000500
Epoch:[1/1](200/5000) loss:2.453 lr:0.000499  # 几乎不变
```

**排查步骤**：

```python

# 2. 检查学习率
print(f"当前学习率: {optimizer.param_groups[0]['lr']}")

# 3. 检查梯度
for name, param in model.named_parameters():
    if param.grad is not None:
        print(f"{name}: grad_mean={param.grad.mean():.6f}, grad_max={param.grad.max():.6f}")

# 4. 调整学习率
# 如果 loss 很稳定，尝试提高学习率
parser.add_argument("--learning_rate", type=float, default=1e-3)  # 提高到1e-3

# 5. 检查损失函数
# 确保使用了正确的 reduction
loss_fct = nn.CrossEntropyLoss(reduction="mean")  # 而非 sum
```

## 推理阶段问题

### ❌ 问题：输出重复

**现象**：
```
用户: 你好
模型: 你你好你好你好你好你好你好你好...
```

**解决方案**：

```python
# 1. 调整采样参数
generated = model.generate(
    **inputs,
    temperature=0.8,        # 增加（从0.1到0.8）
    top_p=0.9,
    do_sample=True,           # 启用采样
    repetition_penalty=1.1,   # 添加重复惩罚
)

# 2. 检查训练数据质量
# 确保训练数据没有重复内容

# 3. 调整 max_new_tokens
# 防止生成过长导致重复
generated = model.generate(
    **inputs,
    max_new_tokens=128,  # 限制最大长度
)
```

### ❌ 问题：输出不完整

**现象**：
```
用户: 请介绍一下人工智能

```

**解决方案**：

```python
# 1. 增加 max_new_tokens
generated = model.generate(
    **inputs,
    max_new_tokens=512,  # 从128增加到512
)

# 2. 调整停止条件

# 3. 检查位置编码
```

### ❌ 问题：JSON 格式错误

**现象**：

```python
)

# 方案2: 使用约束解码

intent_schema = {
    "type": "object",
    "properties": {
        "intent": {"type": "string"},
        "slots": {"type": "object"}
    }
}

generator = outlines.generate.json(model, intent_schema)
result = generator("查一下明天北京的天气")
# 保证输出合法 JSON

# 方案3: 后处理重试
def robust_json_parse(model, user_input, max_retries=3):
```

## 训练中断恢复

### 问题：训练中途崩溃

**解决方案**：

```python
# 1. 启用自动保存
parser.add_argument("--save_interval", type=int, default=100)

# 2. 检查点恢复
def load_checkpoint(model, optimizer, checkpoint_path):
    checkpoint = torch.load(checkpoint_path)
    model.load_state_dict(checkpoint['model_state_dict'])
    optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
    start_epoch = checkpoint['epoch']
    return start_epoch

# 3. 使用训练脚本
python train_pretrain.py --resume ../out/checkpoint_epoch_5.pth
```

## 常见错误代码

| 错误代码 | 原因 | 解决方案 |
|---------|------|---------|
| `RuntimeError: CUDA out of memory` | 显存不足 | 减小 batch_size 或 max_seq_len |
| `ValueError: Token index out of range` | 词表不匹配 | 检查 tokenizer 路径 |
| `RuntimeError: mat1 and mat2 shapes cannot be multiplied` | 维度不匹配 | 检查 hidden_size 配置 |
| `FileNotFoundError: model/*.pth` | 权重文件不存在 | 先训练预模型或下载权重 |
| `KeyError: 'conversations'` | 数据格式错误 | 检查数据格式是否正确 |
| `loss: nan` | 训练不稳定 | 降低学习率、检查数据 |
