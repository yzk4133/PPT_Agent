## 两层准备策略

### 第一层：八股文（必须背熟）

| 类别 | 必答知识点 | 面试官关注点 |
|------|-----------|-------------|
| **训练流程** | Pretrain → SFT → LoRA 三阶段 | 是否理解每阶段作用 |
| **LoRA 原理** | rank、alpha、target_modules | 参数量减少 90% 的原因 |
| **Loss 计算** | cross-entropy、loss_mask | 只在 assistant 部分计算 |
| **评估指标** | 准确率、验证损失、过拟合检测 | 任务适配性（生成 vs 分类） |
| **数据质量** | JSON 格式验证、字段约束、人工抽检 | 工程落地能力 |

**30 秒答案 = 八股文的精简版**，需要背到脱口而出。

---

### 第二层：训练细节（深挖加分）

| 问题类型 | 示例 | 应对策略 |
|---------|------|---------|
| **参数调优** | "为什么 learning rate 选 1e-4？" | 解释 LoRA 标准配置，实验发现 1e-4 稳定 |
| **异常处理** | "如果训练不收敛怎么办？" | 降低学习率、检查数据质量、增加 warmup |
| **架构选择** | "为什么不用全量微调？" | 显存成本、训练速度、adapter 灵活性 |
| **数据策略** | "数据多了/少了会怎样？" | 看验证损失曲线、过拟合检测 |

**这些需要实际训练经验**，纯背不够。

## 脑中模拟框架（5步走）

### Step 1: 任务定义

```
Q: 为什么要做？
A: 识别用户 PPT 需求 → 结构化 JSON

Q: 为什么不用大模型？
A: 学习 + 延迟/成本优化

Q: 任务本质是什么？
A: NLG 生成任务，不是分类
```

**关键指标**：延迟 20ms vs 1200ms（60x），成本 $0.001 vs $0.02（20x）

---

### Step 2: 数据准备

```
Q: 数据从哪来？
A: 50 条真实 + GPT-4 生成 = 850 条

Q: 为什么 850 条？
A: MVP 验证，LoRA 参数少够用，推荐 200-1000 条

Q: 怎么保证质量？
A: JSON 格式验证 + 人工抽检

Q: 多了/少了会怎样？
A: <50 过拟合，>5000 收益递减

Q: 怎么判断数据量够不够？
A: 看验证损失曲线、训练/验证损失差距、实际准确率
```

**关键指标**：字段级别准确率 92%（topic 95%、type 90%、style 89%）

---

### Step 3: 模型选择

```
Q: 为什么 MiniMind？
A: 学习友好 + 训练成本低 + 中文优化

Q: 为什么 25.8M 参数？
A: size ablation 实验，性价比拐点
   - 10M: 85% 准确率（不够）
   - 25.8M: 92% 准确率（合适）
   - 60M: 94% 准确率（收益递减）

Q: 为什么 LoRA？
A: 参数少 90% + 训练快 6 小时 + adapter 灵活

Q: LoRA 参数量怎么算？
A: rank × (hidden_size × 2 + hidden_size × 2) × num_layers
   = 8 × (512 × 2 + 512 × 2) × 8 ≈ 131K
```

**关键参数**：rank=8, alpha=32, dropout=0.1

---

### Step 4: 训练过程

```
Q: Loss 怎么算？
A: cross-entropy，逐 token 预测下一个 token
   只在 assistant 回复部分计算（用户输入不计算）

Q: 为什么 lr=1e-4？
A: LoRA 标准配置，实验验证稳定
   太高（>1e-3）不稳定，太低（<1e-5）收敛慢

Q: 训练多久？
A: 10 轮，损失从 2.5 降到 0.3

Q: 怎么判断收敛？
A: 验证损失持平 + 训练/验证损失差距小（<0.1）

Q: batch size 怎么选？
A: 32，显存允许的情况下尽可能大
```

**关键指标**：验证损失 0.315（Epoch 9），训练 6 小时

---

### Step 5: 评估与部署

```
Q: 用什么指标？
A: 字段级别准确率（生成任务不是分类，不用 F1）

Q: 效果怎么样？
A: 92% vs 大模型 97%，差 5 个百分点
   换来 60 倍速度提升 + 20 倍成本降低

Q: 怎么兜底？
A: 置信度 < 0.7 fallback 到大模型

Q: 怎么监控线上效果？
A: 定期抽检 100 条 + 统计 fallback 率
```

**关键指标**：字段准确率（topic 95%、type 90%、style 89%、整体 92%）

## 学习路径建议

### 第 1 周：理解原理

- [ ] 看 LoRA 论文（10 页重点）
- [ ] 理解 attention、GQA、SwiGLU
- [ ] 背熟 MiniMind 架构参数
  - hidden_size=512, num_layers=8, vocab_size=6400
  - GQA: num_heads=8, num_kv_heads=2

**检查点**：能不看文档画出 MiniMind 架构图

---

### 第 2 周：跑通训练

- [ ] Pretrain → SFT → LoRA 三阶段都跑一遍
- [ ] 观察损失曲线，理解"收敛"是什么样
- [ ] 尝试改参数（lr、batch_size、rank）
- [ ] 记录实验结果

**检查点**：能独立解释为什么某个参数选择某个值

---

### 第 3 周：准备八股

- [ ] 把每个问题写成 150-250 字（参考 QuickRef.md）
- [ ] 对着镜子练 30 秒说完
- [ ] 准备"追问"答案（见下文）
- [ ] 录音自查，改进表达

**检查点**：闭眼睛能从头到尾讲完整个故事

---

### 第 4 周：模拟面试

- [ ] 找人模拟 / 录音自查
- [ ] 重点练：卡壳的地方、逻辑不清的地方
- [ ] 准备 3 个"亮点"故事
  1. "我发现数据问题后如何解决"
  2. "我如何优化训练速度"
  3. "我如何设计兜底机制"

**检查点**：任何追问都能在 5 秒内开始回答

## 常见追问预警

| 八股答案 | 可能的追问 | 准备方向 |
|---------|-----------|---------|
| "用 LoRA 因为参数少" | "LoRA 参数量怎么算？" | rank × (hidden_size × 2 + hidden_size × 2) × num_layers |
| "准确率 92%" | "测试集怎么分的？" | 随机划分 8:1:1，保证分布一致 |
| "用 cross-entropy loss" | "为什么不用 focal loss？" | 数据均衡不需要 focal，简单任务足够 |
| "数据 850 条" | "会不会过拟合？" | LoRA 参数少、看验证损失、必要时扩充 |
| "loss 从 2.5 降到 0.3" | "训练损失还是验证损失？" | 两者都要看，验证损失更关键 |
| "fallback 到大模型" | "置信度怎么算？" | 输出概率分布的熵 / max probability |
| "训练 10 轮" | "为什么不是 5 轮或 20 轮？" | 实验：10 轮验证损失持平，再多过拟合 |
| "batch size 32" | "为什么不是 16 或 64？" | 显存限制下尽可能大，太大收敛慢 |

## 实际准备建议

### 每天 30 分钟，分三块

1. **10 分钟**：背 2-3 个八股答案（速读 5 遍）
2. **10 分钟**：模拟回答 + 录音，检查卡壳
3. **10 分钟**：准备 1-2 个追问的答案

### 关键原则

| 原则 | 说明 |
|------|------|
| **八股要背熟** | 30 秒脱口而出，不要犹豫 |
| **细节要理解** | 能解释为什么，不能只说结果 |
| **数据要真实** | 不要编造，不确定就说没试过 |
| **逻辑要连贯** | 整个故事从头到尾能串起来 |

### 面试技巧

**被问住时的应对**：
```
"这个问题我当时确实没有深入验证，不过从原理来说...
如果让我现在设计，我会先做 X 实验，观察 Y 指标...
"
```

**展示思考过程**：
```
"我当时的考虑是...，后来发现...，所以调整为...
这个过程让我学到了...
"
```

## 面试官的追问套路

### 套路 1：验证理解

| 你说的 | 面试官追问 | 背后逻辑 |
|-------|-----------|---------|
| "用 LoRA" | "LoRA 参数量怎么算？" | 验证是否真的理解 |
| "85 条数据" | "会不会过拟合？" | 检查工程经验 |
| "准确率 92%" | "测试集怎么分的？" | 验证实验严谨性 |
| "loss 从 2.5 降到 0.3" | "训练损失还是验证损失？" | 细节控会问 |
| "fallback 到大模型" | "置信度怎么算？" | 实现细节 |

### 套路 2：边界测试

| 问法 | 意图 | 应对 |
|------|------|------|
| "如果数据扩大 10 倍会怎样？" | 测试系统思维 | 收益递减、可能需要调整 rank |
| "如果换成 7B 模型会怎样？" | 测试迁移能力 | LoRA 仍然适用，rank 可能要调大 |
| "如果任务改成分类会怎样？" | 测试任务理解 | 改用 classification head、用 F1 指标 |

### 套路 3：深度挖掘

| 问法 | 意图 | 应对 |
|------|------|------|
| "你最满意的优化是什么？" | 亮点挖掘 | 准备好具体案例和数据 |
| "如果重来一次会怎么改进？" | 反思能力 | 诚实说出不足 + 改进方向 |
