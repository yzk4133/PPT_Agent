## 1. 整体架构概览

### 1.1 直观理解

```
MiniMind 的工作流程：

输入文本
  "做一个关于人工智能的PPT"
    ↓
[Tokenizer] 分词
  ["做", "一个", "关于", "人工", "智能", "的", "P", "P", "T"]
    ↓
[Embedding] 转为向量
  [[0.1, -0.2, ...], [0.3, 0.1, ...], ...]  # 9 × 512
    ↓
[Transformer Block × 8] 逐层处理
  每一层都增强表示
    ↓
[LM Head] 预测下一个词
  输出每个词的概率
    ↓
生成 JSON
  {"topic": "人工智能", "style": "科技感"}
```

### 1.2 架构图

```
┌─────────────────────────────────────────────────────────┐
│                    MiniMind2-Small                      │
│                    (26M Parameters)                     │
└─────────────────────────────────────────────────────────┘

输入: "做一个关于人工智能的PPT" (9 tokens)
  │
  ▼
┌─────────────────────────────────────────────────────┐
│  Embedding 层 (3.3M 参数)                             │
│  - Token Embedding: 6400 × 512                       │
│  作用: 将离散的 token ID 转为连续向量                │
└─────────────────────────────────────────────────────┘
  │ 输出: (9, 512)
  ▼
┌─────────────────────────────────────────────────────┐
│  Transformer Block 1                                │
│  ├─ RMSNorm (归一化)                                 │
│  ├─ RoPE (位置编码)                                  │
│  ├─ Attention (自注意力)                             │
│  ├─ RMSNorm (归一化)                                 │
│  └─ FeedForward (前馈网络)                           │
├─────────────────────────────────────────────────────┤
│  Transformer Block 2                                │
│  └─ ... (同样的结构)                                 │
├─────────────────────────────────────────────────────┤
│  ...                                                │
├─────────────────────────────────────────────────────┤
│  Transformer Block 8                                │
│  └─ ... (同样的结构)                                 │
└─────────────────────────────────────────────────────┘
  │ 输出: (9, 512)
  ▼
┌─────────────────────────────────────────────────────┐
│  LM Head (3.3M 参数，与 Embedding 共享权重)           │

│  作用: 预测下一个 token                              │
└─────────────────────────────────────────────────────┘
  │ 输出: (9, 6400) - 每个位置对词表的概率分布
  ▼

```

### 1.3 核心概念

```
Tensor (张量) 的维度变化：

输入: (batch_size, seq_len) = (1, 9)
      ↓ Embedding
     (1, 9, 512)
      ↓ Transformer Block × 8
     (1, 9, 512)
      ↓ LM Head
     (1, 9, 6400)

含义：
  - batch_size=1: 处理 1 个样本
  - seq_len=9: 序列长度 9 个 token
  - 512: 每个 token 用 512 维向量表示
  - 6400: 词表大小，输出对每个词的概率
```

## 2. Embedding 层：将词转为向量

### 2.1 为什么需要 Embedding？

```
问题：计算机如何理解"词"？

离散表示（One-hot）:
  "人工智能" = [0, 0, ..., 1, ..., 0]  # 第 1234 位为 1
  ❌ 维度太高（6400 维）
  ❌ 稀疏（只有一个 1）
  ❌ 无法表示词之间的关系

连续表示（Embedding）:
  "人工智能" = [0.1, -0.2, 0.5, ..., 0.3]  # 512 维
  ✅ 维度适中（512 维）
  ✅ 密集（所有维度都有信息）
  ✅ 可以表示词之间的关系（相似的词向量相近）
```

### 2.2 Embedding 的原理

```
直观理解：

Embedding 是一个"查表"操作：

词表:
  ID 0: "做"     → [0.1, 0.2, ...]
  ID 1: "一个"   → [0.3, -0.1, ...]
  ID 2: "关于"   → [-0.2, 0.4, ...]
  ...
  ID 6399: "T"   → [0.5, -0.3, ...]

Embedding 层就是一个大矩阵：
  shape: (6400, 512)
  第 i 行 = 第 i 个词的向量表示

操作：
  对于输入序列 [123, 456, 789, ...]
  从 Embedding 矩阵中取出第 123, 456, 789 行
  组合成 (seq_len, 512) 的矩阵
```

### 2.3 数学表示

```
Embedding 操作：

给定：
  - 词表大小 V = 6400
  - Embedding 维度 d = 512
  - 输入序列 x = [x₁, x₂, ..., xₙ]，其中 xᵢ ∈ [0, V)

Embedding 矩阵:
  E ∈ R^(V × d)

操作：
  emb(x) = E[x] ∈ R^(n × d)

其中 E[x] 表示从 E 中取出第 x₁, x₂, ..., xₙ 行

例子：
  x = [123, 456, 789]
  emb(x) = E[[123, 456, 789]]
        = [E[123], E[456], E[789]]
        = [[0.1, 0.2, ...],  # 123 对应的行
           [0.3, -0.1, ...],  # 456 对应的行
           [-0.2, 0.4, ...]]  # 789 对应的行
        ∈ R^(3 × 512)
```

### 2.4 为什么 Embedding 能学习语义？

```
直观理解：

训练过程中，模型会学习到：

1. 相似的词 → 相近的向量
   "人工智能" ≈ "AI" ≈ "机器学习"
   向量: [0.1, 0.2, ...] ≈ [0.12, 0.19, ...] ≈ [0.08, 0.21, ...]

2. 相关的词 → 有规律的向量
   "国王" - "男人" ≈ "女王" - "女人"
   向量运算: king - man ≈ queen - woman

3. 语法功能相似的词 → 在向量空间中聚集
   所有动词在一个区域，所有名词在另一个区域

这是如何做到的？

因为模型在预测下一个词时，需要理解词之间的关系，
训练过程中自动学习到了这些语义信息。
```

### 2.5 参数量计算

```
Embedding 层参数：

Token Embedding:
  参数量 = vocab_size × hidden_size
         = 6400 × 512
         = 3,276,800
         ≈ 3.3M

Position Embedding (如果用绝对位置编码):
  参数量 = max_seq_len × hidden_size
         = 32768 × 512
         = 16,777,216
         ≈ 16.8M

但我们用 RoPE，不需要 Position Embedding！
  参数量 = 0

MiniMind 使用 RoPE，所以 Embedding 层只有 3.3M 参数
```

### 2.6 代码实现

```python
import torch
import torch.nn as nn

class TokenEmbedding(nn.Module):
    """Token Embedding 层"""

    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        # Embedding 矩阵：vocab_size × hidden_size
        self.embedding = nn.Embedding(vocab_size, hidden_size)

    def forward(self, input_ids):
        """
        参数:
          input_ids: (batch_size, seq_len)

        返回:
          embeddings: (batch_size, seq_len, hidden_size)
        """
        # 从 Embedding 矩阵中查找
        embeddings = self.embedding(input_ids)
        return embeddings

# 使用示例
vocab_size = 6400
hidden_size = 512

# 创建 Embedding 层
token_emb = TokenEmbedding(vocab_size, hidden_size)

# 输入：3 个 token ID
input_ids = torch.tensor([[123, 456, 789]])  # (1, 3)

# Embedding
embeddings = token_emb(input_ids)  # (1, 3, 512)

print(embeddings.shape)  # torch.Size([1, 3, 512])
```

## 3. Transformer Block：核心计算单元

### 3.1 整体结构

```
Transformer Block 的组成：

输入 x (batch_size, seq_len, hidden_size)
  │
  ├─┬─ RMSNorm ─────────────────────────┐
  │ │                                    │
  │ ▼                                    │
  │ RoPE (注入位置信息)                   │
  │ │                                    │
  │ ▼                                    │
  │ Attention (自注意力)                  │
  │ │                                    │
  │ ▼                                    │
  │ Dropout                               │
  │ │                                    │
  ├─┴────────────────────────────────────┤
  │ 残差连接: x + Attention(x)           │
  ├─┬────────────────────────────────────┤
  │ │                                    │
  │ ▼                                    │
  │ RMSNorm                               │
  │ │                                    │
  │ ▼                                    │
  │ FeedForward (前馈网络)                │
  │ │                                    │
  │ ▼                                    │
  │ Dropout                               │
  │ │                                    │
  ├─┴────────────────────────────────────┤
  │ 残差连接: x + FeedForward(x)         │
  └────────────────────────────────────────┘
输出 (batch_size, seq_len, hidden_size)
```

### 3.2 为什么需要残差连接？

```
问题：深层网络难以训练

原因：
  - 反向传播时，梯度逐层衰减
  - 深层网络容易梯度消失
  - 难以训练 8 层以上的网络

解决方案：残差连接 (Residual Connection)

x → [Layer] → + → 输出
     ↑          │
     └──────────┘

公式：output = x + Layer(x)

效果：
  - 提供梯度的"高速公路"
  - 即使 Layer(x) 很小，梯度也能直接传播
  - 可以训练很深的网络（100+ 层）
```

### 3.3 为什么先 Norm 后 Attention？

```
两种顺序：

Pre-Norm (我们用的):
  RMSNorm → Attention → 残差连接
  RMSNorm → FeedForward → 残差连接

Post-Norm:
  Attention → RMSNorm → 残差连接
  FeedForward → RMSNorm → 残差连接

为什么 Pre-Norm 更好？

1. 训练稳定性：
   Pre-Norm: 每层输入都归一化，梯度稳定
   Post-Norm: 深层可能累积误差

2. 收敛速度：
   Pre-Norm: 收敛更快
   Post-Norm: 收敛较慢

现代 Transformer 都用 Pre-Norm
```

### 3.4 代码框架

```python
class TransformerBlock(nn.Module):
    """Transformer Block"""

    def __init__(self, config):
        super().__init__()
        # Attention 子层
        self.attention = Attention(config)
        self.norm1 = RMSNorm(config.hidden_size)
        self.dropout1 = nn.Dropout(config.dropout)

        # FeedForward 子层
        self.feed_forward = FeedForward(config)
        self.norm2 = RMSNorm(config.hidden_size)
        self.dropout2 = nn.Dropout(config.dropout)

    def forward(self, x, use_cache=False):
        """
        参数:
          x: (batch_size, seq_len, hidden_size)
          use_cache: 是否使用 KV Cache

        返回:
          output: (batch_size, seq_len, hidden_size)
          cache: (可选) KV Cache
        """
        # Attention 子层（Pre-Norm）
        residual = x
        x = self.norm1(x)
        x, cache = self.attention(x, use_cache=use_cache)
        x = self.dropout1(x)
        x = x + residual  # 残差连接

        # FeedForward 子层（Pre-Norm）
        residual = x
        x = self.norm2(x)
        x = self.feed_forward(x)
        x = self.dropout2(x)
        x = x + residual  # 残差连接

        return x, cache
```

## 4. Attention：让模型"关注"重要信息

### 4.1 直观理解

```
Attention 的作用：

句子："小明的狗因为太兴奋，所以它一直在叫"

问题："它"指代什么？

人类理解：
  "它" → 看上下文 → "狗"（不是"小明"）

Attention 机制：
  计算"它"与每个词的相关性
  "它"与"狗"的相关性高
  "它"与"小明"的相关性低
  → "它"主要关注"狗"

可视化：
  it  ←───────┐
  小明  0.1   │
  的    0.1   │  注意力权重
  狗    0.6   │
  因为  0.05  │
  太    0.05  │
  兴奋  0.05  │
  所以  0.03  │
  一直  0.01  │
  在    0.01  ┘
  叫
```

### 4.2 数学原理

#### 4.2.1 Scaled Dot-Product Attention

```
核心公式：

Attention(Q, K, V) = softmax(QK^T / √d_k) V

其中：
  Q (Query): 查询向量，"我想要什么"
  K (Key): 键向量，"我能提供什么"
  V (Value): 值向量，"我的内容是什么"
  d_k: Key 的维度，用于缩放

直观理解：

1. QK^T: 计算 Query 和所有 Key 的相似度
   分数高 = 相关性强

2. / √d_k: 缩放，防止分数过大导致 softmax 饱和

3. softmax: 将分数转为概率分布（和为 1）

4. × V: 根据概率对 Value 加权求和
```

#### 4.2.2 Multi-Head Attention

```

为什么需要 Multi-Head？

单头 Attention 的问题：
  - 只能学习一种关系模式
  - 表达能力有限

Multi-Head Attention：

使用 h 个不同的 Attention 头，每个头学习不同的关系模式

公式：
  MultiHead(Q, K, V) = Concat(head₁, ..., headₕ)W^O

其中：
  headᵢ = Attention(QWᵢ_Q, KWᵢ_K, VWᵢ_V)
  Wᵢ_Q, Wᵢ_K, Wᵢ_V: 第 i 个头的投影矩阵
  W^O: 输出投影矩阵

直观例子：

句子："苹果发布了新产品"

不同头关注不同的关系：
  Head 1: "苹果" ↔ "发布" (主谓关系)
  Head 2: "苹果" ↔ "产品" (定宾关系)
  Head 3: "发布" ↔ "产品" (动宾关系)
  ...
```

#### 4.2.3 GQA (Grouped Query Attention)

```

问题：标准 MHA 的 Key 和 Value 太大

MHA (8 头):
  Q: 8 头
  K: 8 头
  V: 8 头
  参数量：4 × d²

GQA (8 头 Q, 2 头 KV):
  Q: 8 头
  K: 2 头（每 4 个 Q 头共享 1 组 KV）
  V: 2 头
  参数量：4 × d² × (1 + 3/4) ≈ 2.5 × d²

原理：
  - 将 Q 头分组，每组共享一组 KV
  - 减少参数量（37%）
  - 减少 KV Cache（75%）
  - 性能几乎无损（-0.5%）

可视化：

MHA:
  Q₁ → K₁, V₁
  Q₂ → K₂, V₂
  ...
  Q₈ → K₈, V₈

GQA (4组):
  Q₁,Q₂ → K₁, V₁
  Q₃,Q₄ → K₁, V₁
  Q₅,Q₆ → K₂, V₂
  Q₇,Q₈ → K₂, V₂
```

### 4.3 计算流程

```

Attention 的计算流程：

输入: x (batch_size, seq_len, hidden_size)

1. 投影到 Q, K, V:
   Q = x @ W_Q  # (batch_size, seq_len, head_dim * num_heads)
   K = x @ W_K
   V = x @ W_V

2. 分割成多个头:
   Q = Q.view(batch_size, seq_len, num_heads, head_dim)
   K = K.view(batch_size, seq_len, num_kv_heads, head_dim)
   V = V.view(batch_size, seq_len, num_kv_heads, head_dim)

3. 转置以便矩阵乘法:
   Q = Q.transpose(1, 2)  # (batch_size, num_heads, seq_len, head_dim)
   K = K.transpose(1, 2)
   V = V.transpose(1, 2)

4. 计算注意力分数:
   scores = Q @ K.transpose(-2, -1)  # (batch_size, num_heads, seq_len, seq_len)
   scores = scores / √head_dim

5. 应用因果掩码（只看前面）:
   mask = torch.tril(torch.ones(seq_len, seq_len))
   scores = scores.masked_fill(mask == 0, -inf)

6. Softmax:
   attn_weights = softmax(scores, dim=-1)

7. 加权求和:
   output = attn_weights @ V  # (batch_size, num_heads, seq_len, head_dim)

8. 合并多头:
   output = output.transpose(1, 2)  # (batch_size, seq_len, num_heads, head_dim)
   output = output.reshape(batch_size, seq_len, hidden_size)

9. 输出投影:
   output = output @ W_O  # (batch_size, seq_len, hidden_size)
```

### 4.4 因果掩码

```

为什么需要因果掩码？

因为我们是 Decoder-only，生成时是自回归的：
  - 生成第 1 个词时，只能看到第 1 个词
  - 生成第 2 个词时，只能看到第 1、2 个词
  - 生成第 n 个词时，只能看到前 n 个词

训练时也要模拟这个限制：

scores 矩阵 (seq_len=4):

     位置 1   位置 2   位置 3   位置 4
位置 1  ✓       ✗        ✗        ✗
位置 2  ✓       ✓        ✗        ✗
位置 3  ✓       ✓        ✓        ✗
位置 4  ✓       ✓        ✓        ✓

✓ = 可以看（计算注意力）
✗ = 不能看（ masked）

实现：
  mask = torch.tril(torch.ones(seq_len, seq_len))
  scores = scores.masked_fill(mask == 0, float('-inf'))
```

### 4.5 参数量计算

```

Attention 层的参数：

对于 hidden_size=512, num_heads=8, num_kv_heads=2:

每个头的维度:
  head_dim = hidden_size / num_heads = 512 / 8 = 64

Q_proj:
  hidden_size × hidden_size = 512 × 512 = 262K

K_proj:
  hidden_size × (head_dim × num_kv_heads)
  = 512 × (64 × 2)
  = 512 × 128
  = 65K

V_proj:
  同 K_proj = 65K

O_proj:
  hidden_size × hidden_size = 512 × 512 = 262K

总计: 262K + 65K + 65K + 262K = 654K ≈ 0.65M

对比 MHA (8 头 Q, 8 头 K, 8 头 V):
  总计: 262K × 4 = 1,048K ≈ 1.0M

节省: (1.0M - 0.65M) / 1.0M = 37%
```

### 4.6 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class Attention(nn.Module):
    """GQA Attention 机制"""

    def __init__(self, config):
        super().__init__()
        self.num_heads = config.num_attention_heads       # 8
        self.num_kv_heads = config.num_key_value_heads   # 2
        self.head_dim = config.hidden_size // self.num_heads  # 64
        self.hidden_size = config.hidden_size             # 512

        # Q, K, V 投影
        self.q_proj = nn.Linear(self.hidden_size, self.num_heads * self.head_dim, bias=False)
        self.k_proj = nn.Linear(self.hidden_size, self.num_kv_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(self.hidden_size, self.num_kv_heads * self.head_dim, bias=False)

        # 输出投影
        self.o_proj = nn.Linear(self.num_heads * self.head_dim, self.hidden_size, bias=False)

        # Dropout
        self.dropout = nn.Dropout(config.dropout)

    def forward(self, x, use_cache=False):
        """
        参数:
          x: (batch_size, seq_len, hidden_size)
          use_cache: 是否使用 KV Cache

        返回:
          output: (batch_size, seq_len, hidden_size)
          cache: (可选) KV Cache
        """
        batch_size, seq_len, _ = x.shape

        # 计算 Q, K, V
        Q = self.q_proj(x)  # (batch_size, seq_len, num_heads * head_dim)
        K = self.k_proj(x)  # (batch_size, seq_len, num_kv_heads * head_dim)
        V = self.v_proj(x)  # (batch_size, seq_len, num_kv_heads * head_dim)

        # 分割成多头
        Q = Q.view(batch_size, seq_len, self.num_heads, self.head_dim)
        K = K.view(batch_size, seq_len, self.num_kv_heads, self.head_dim)
        V = V.view(batch_size, seq_len, self.num_kv_heads, self.head_dim)

        # 转置: (batch_size, num_heads, seq_len, head_dim)
        Q = Q.transpose(1, 2)
        K = K.transpose(1, 2)
        V = V.transpose(1, 2)

        # 计算 Attention 分数
        scores = torch.matmul(Q, K.transpose(-2, -1))  # (batch_size, num_heads, seq_len, seq_len)
        scores = scores / math.sqrt(self.head_dim)      # 缩放

        # 因果掩码
        causal_mask = torch.tril(torch.ones(seq_len, seq_len, device=x.device)).bool()
        scores = scores.masked_fill(~causal_mask, float('-inf'))

        # Softmax
        attn_weights = F.softmax(scores.float(), dim=-1).type_as(Q)

        # Dropout
        attn_weights = self.dropout(attn_weights)

        # 加权求和
        # 注意：需要处理 GQA 的头数不匹配
        if self.num_kv_heads != self.num_heads:
            # 扩展 K 和 V 以匹配 Q 的头数
            K = K.repeat_interleave(self.num_heads // self.num_kv_heads, dim=1)
            V = V.repeat_interleave(self.num_heads // self.num_kv_heads, dim=1)

        output = torch.matmul(attn_weights, V)  # (batch_size, num_heads, seq_len, head_dim)

        # 合并多头
        output = output.transpose(1, 2)  # (batch_size, seq_len, num_heads, head_dim)
        output = output.reshape(batch_size, seq_len, self.hidden_size)

        # 输出投影
        output = self.o_proj(output)

        return output, None
```

## 5. FeedForward：增加模型表达能力

### 5.1 直观理解

```

FeedForward 的作用：

Attention 已经混合了不同位置的信息，
FeedForward 进一步处理每个位置的信息。

类比：
  Attention = 团队讨论（交换信息）
  FeedForward = 个人思考（处理信息）

结构：
  x → [Gate] ─┬─→ [×] ─→ [Down] ─→ 输出
            │
         [Up] ─┘

SwiGLU 激活：
  - Gate 门控控制信息流
  - 更好的梯度传播
```

### 5.2 SwiGLU 激活函数

```

SwiGLU 的公式：

FFN(x) = (xW₁ ⊙ σ(xW₂))W₃

其中：
  ⊙: 逐元素乘法
  σ: sigmoid 激活函数
  W₁, W₂, W₃: 权重矩阵

步骤分解：

1. 上投影 (up_proj):
   gate = xW₁
   up = xW₂

2. 门控激活:
   gated = gate ⊙ σ(up)

3. 下投影 (down_proj):
   output = gatedW₃

相比 ReLU 的优势：

ReLU FFN:
  FFN(x) = ReLU(xW₁)W₂
  - 简单但表达能力有限
  - 容易梯度消失

SwiGLU FFN:
  FFN(x) = (xW₁ ⊙ σ(xW₂))W₃
  - 门控机制（类似 LSTM）
  - 更好的梯度传播
  - 性能提升约 7%
```

### 5.3 参数量计算

```

FeedForward 层的参数：

对于 hidden_size=512, expansion_factor=8/3≈1.365:

中间维度:
  intermediate_size = hidden_size × expansion_factor
                    = 512 × 8/3
                    ≈ 1365

gate_proj:
  hidden_size × intermediate_size = 512 × 1365 = 700K

up_proj:
  同上 = 700K

down_proj:
  intermediate_size × hidden_size = 1365 × 512 = 700K

总计: 700K × 3 = 2.1M

对比 ReLU FFN (4× hidden_size):
  总计: 512 × 2048 × 2 = 2.0M

SwiGLU 参数略多，但性能更好
```

### 5.4 代码实现

```python
class FeedForward(nn.Module):
    """SwiGLU FeedForward 网络"""

    def __init__(self, config):
        super().__init__()
        self.hidden_size = config.hidden_size
        intermediate_size = int(config.hidden_size * config.ffn_multiplier)  # 8/3

        # 三个线性层
        self.gate_proj = nn.Linear(self.hidden_size, intermediate_size, bias=False)
        self.up_proj = nn.Linear(self.hidden_size, intermediate_size, bias=False)
        self.down_proj = nn.Linear(intermediate_size, self.hidden_size, bias=False)

        # Dropout
        self.dropout = nn.Dropout(config.dropout)

    def forward(self, x):
        """
        参数:
          x: (batch_size, seq_len, hidden_size)

        返回:
          output: (batch_size, seq_len, hidden_size)
        """
        # SwiGLU 激活
        gate = self.gate_proj(x)
        up = self.up_proj(x)
        gated = gate * torch.sigmoid(up)  # 逐元素乘法

        # 下投影
        output = self.down_proj(gated)
        output = self.dropout(output)

        return output
```

## 6. RoPE：注入位置信息

### 6.1 为什么需要位置编码？

```

问题：Transformer 本身没有位置概念

示例：
  "我爱你" 和 "你爱我"
  词相同，但意思完全不同

如果没有位置编码：
  Attention 只看到词的语义
  看不到词的位置关系
  → 无法区分词序

解决方案：注入位置信息
```

### 6.2 RoPE 的原理

```

核心思想：通过旋转变换编码相对位置

直观理解：

1. 将 2D 向量 (x, y) 旋转 θ 角:
   x' = x cosθ - y sinθ
   y' = x sinθ + y cosθ

2. 对于 Query 和 Key 向量：
   - Query 向量旋转 m×θ (m 是位置)
   - Key 向量旋转 n×θ (n 是位置)
   - 相对位置 = m - n

3. Attention(Q_m, K_n) 只依赖于 (m - n)
   → 学习相对位置关系
   → 可以外推到更长的序列

数学公式：

给定位置 m 的 Query 向量 q 和位置 n 的 Key 向量 k：

旋转后的内积：
  ⟨q_m, k_n⟩ = ⟨R_θ,m q, R_θ,n k⟩

其中 R_θ,m 是旋转矩阵：
  R_θ,m = [[cos(mθ), -sin(mθ)],
           [sin(mθ),  cos(mθ)]]

关键性质：
  ⟨R_θ,m q, R_θ,n k⟩ = ⟨q, R_θ,(n-m) k⟩
  → 只依赖于相对位置 (n-m)
```

### 6.3 为什么 RoPE 能外推？

```

绝对位置编码 (APE) 的问题：

训练长度：512
测试长度：1024

位置 513 的编码在训练时从未见过！
→ 性能下降

RoPE 的优势：

训练时看到的是相对位置差：
  - 位置 513 和 514 的差 = 1（训练时见过）
  - 位置 513 和 614 的差 = 101（训练时见过）

只要相对位置差在训练范围内，就能外推！

实验结果：
  APE: 长度 1024 时准确率下降 > 5%
  RoPE: 长度 1024 时准确率下降 < 2%
```

### 6.4 代码实现

```python
class RoPE(nn.Module):
    """旋转位置编码"""

    def __init__(self, head_dim, max_seq_len=32768, theta=10000.0):
        super().__init__()
        self.head_dim = head_dim
        self.max_seq_len = max_seq_len
        self.theta = theta

        # 预计算频率
        inv_freq = 1.0 / (theta ** (torch.arange(0, head_dim, 2).float() / head_dim))
        self.register_buffer('inv_freq', inv_freq)

    def forward(self, x, seq_len):
        """
        参数:
          x: (batch_size, num_heads, seq_len, head_dim)
          seq_len: 序列长度

        返回:
          x: 加入位置信息后的 x
        """
        batch_size, num_heads, _, _ = x.shape

        # 生成位置索引
        t = torch.arange(seq_len, device=x.device).float()

        # 计算旋转角度
        freqs = torch.outer(t, self.inv_freq)  # (seq_len, head_dim/2)

        # 转为复数表示
        freqs = torch.polar(torch.ones_like(freqs), freqs)  # (seq_len, head_dim/2)

        # 应用旋转
        x_complex = torch.view_as_complex(x.float())  # (batch, num_heads, seq_len, head_dim/2)
        x_rotated = x_complex * freqs.unsqueeze(0).unsqueeze(1)

        # 转回实数
        x_out = torch.view_as_real(x_rotated).type_as(x)

        return x_out
```

## 7. RMSNorm：稳定训练

### 7.1 为什么需要归一化？

```

问题：深层网络训练不稳定

原因：
  - 数值可能爆炸或消失
  - 梯度可能过大或过小
  - 不同层的数值尺度不一致

解决方案：归一化（Normalization）

LayerNorm / RMSNorm:
  将每层的输出归一化到稳定的范围
  → 训练更稳定
  → 收敛更快
```

### 7.2 RMSNorm vs LayerNorm

```

LayerNorm:

output = (x - mean(x)) / √(var(x) + ε) × γ + β

步骤：
  1. 计算均值 mean(x)
  2. 计算方差 var(x)
  3. 归一化
  4. 缩放 γ 和偏移 β

RMSNorm:

output = x / RMS(x) × γ

其中 RMS(x) = √(mean(x²) + ε)

步骤：
  1. 计算均方根 RMS(x)
  2. 归一化
  3. 缩放 γ（通常不用 β）

优势：
  - 少一次减法（不需要减均值）
  - 计算更快（约 20%）
  - 性能相当（实验验证）
```

### 7.3 代码实现

```python
class RMSNorm(nn.Module):
    """RMS 归一化"""

    def __init__(self, hidden_size, eps=1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(hidden_size))

    def forward(self, x):
        """
        参数:
          x: (batch_size, seq_len, hidden_size)

        返回:
          output: (batch_size, seq_len, hidden_size)
        """
        # 计算 RMS
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)

        # 归一化并缩放
        output = x / rms * self.weight

        return output
```

class MiniMindForCausalLM(nn.Module):
    """完整的 MiniMind 模型"""

    def __init__(self, config):
        super().__init__()
        self.config = config

        # Embedding
        self.embedding = nn.Embedding(config.vocab_size, config.hidden_size)

        # Transformer Blocks
        self.layers = nn.ModuleList([
            TransformerBlock(config) for _ in range(config.num_hidden_layers)
        ])

        # RMSNorm (最终层)
        self.norm = RMSNorm(config.hidden_size)

        # LM Head
        self.lm_head = nn.Linear(config.hidden_size, config.vocab_size, bias=False)

        # 权重共享：LM Head 和 Embedding 共享权重
        self.lm_head.weight = self.embedding.weight

    def forward(self, input_ids):
        """
        参数:
          input_ids: (batch_size, seq_len)

        返回:
          logits: (batch_size, seq_len, vocab_size)
        """
        x = self.embedding(input_ids)  # (batch_size, seq_len, hidden_size)

        # 通过所有 Transformer Block
        for layer in self.layers:
            x, _ = layer(x)

        # 最终归一化
        x = self.norm(x)

        # LM Head
        logits = self.lm_head(x)  # (batch_size, seq_len, vocab_size)

        return logits
