### 1.2 阶段间的依赖关系

| 阶段 | 前置要求 | 输出权重 | 是否必须 |
|------|----------|----------|----------|
| **预训练** | 无 | `pretrain_512.pth` | ✅ 是 |
| **SFT** | `pretrain_512.pth` | `full_sft_512.pth` | ✅ 是 |
| **LoRA** | `full_sft_512.pth` | `lora_weights.pth` | ❌ 可选 |
| **DPO** | `full_sft_512.pth` | `dpo_512.pth` | ❌ 可选 |
| **推理** | 任意训练阶段权重 | - | - |

**典型使用场景**：

1. **学习研究**：预训练 → SFT → 推理
2. **快速适配**：加载预训练 SFT 权重 → LoRA 微调 → 推理
3. **高质量对齐**：预训练 → SFT → DPO → 推理
4. **资源受限**：跳过预训练，直接使用开源 SFT 权重

## 2. 预训练（Pretraining）

### 2.1 目的和作用

**预训练的目标**：让模型学习语言知识，包括：
- 词汇和语法
- 世界知识
- 推理能力
- 上下文理解

**为什么需要预训练？**

```
随机初始化模型 → 不理解任何语言
                    ↓
              预训练 100B tokens
                    ↓
         理解语言，可以续写文本
```

### 2.2 关键超参数

**位置**：`trainer/train_pretrain.py`

| 参数 | 默认值 | 说明 | 调优建议 |
|------|--------|------|----------|
| `--epochs` | 1 | 训练轮数 | 数据量大时 1 轮足够 |
| `--batch_size` | 32 | 批大小 | 根据 GPU 显存调整 |
| `--learning_rate` | 5e-4 | 学习率 | 小模型可用较大学习率 |
| `--max_seq_len` | 512 | 最大序列长度 | 影响显存和训练速度 |
| `--hidden_size` | 512 | 隐藏层维度 | 决定模型规模 |
| `--num_hidden_layers` | 8 | Transformer 层数 | 决定模型深度 |
| `--accumulation_steps` | 8 | 梯度累积步数 | 等效 batch_size × accumulation |
| `--grad_clip` | 1.0 | 梯度裁剪 | 防止梯度爆炸 |
| `--dtype` | bfloat16 | 数据类型 | bfloat16 更稳定 |

**学习率调度**：Cosine Annealing

```python
# trainer/train_pretrain.py:29-30
def get_lr(current_step, total_steps, lr):
    return lr / 10 + 0.5 * lr * (1 + math.cos(math.pi * current_step / total_steps))
```

**学习率曲线**：
```
lr
│
│    lr/10 + 0.5*lr  ←── 起始学习率
│   ／￣＼
│  ／    ＼
 │ ／      ＼  ←── cos 下降
 │/          ＼
└────────────────────→ step
```

### 2.3 损失函数详解

**标准交叉熵损失**：

```python
# trainer/train_pretrain.py:34-56
loss_fct = nn.CrossEntropyLoss(reduction="none")

# 前向传播
res = model(X)  # X: [batch, seq_len]

# 计算损失
loss = loss_fct(
    res.logits.view(-1, vocab_size),  # [batch*seq_len, vocab_size]
    Y.view(-1)                        # [batch*seq_len]
).view(Y.size())  # [batch, seq_len]

# 应用损失掩码（只计算非 pad 位置）
loss = (loss * loss_mask).sum() / loss_mask.sum()

# 添加 MoE 辅助损失
loss += res.aux_loss

# 梯度累积
loss = loss / args.accumulation_steps
```

**MoE 辅助损失**（当 `use_moe=True` 时）：

```python
# model/model_minimind.py:294-320
# 在 MoE 门控中计算辅助损失
if self.training and self.alpha > 0.0:
    scores_for_aux = scores
    # ... 计算负载均衡损失
    aux_loss = (Pi * fi).sum() * self.alpha
```

**辅助损失的作用**：
- 鼓励各个专家被均匀使用
- 防止某些专家过载，某些专家闲置
- `alpha=0.1` 是辅助损失的权重

**总损失**：
```
Total Loss = CrossEntropy Loss + α × Aux Loss
                    (主损失)        (辅助损失)
```

### 2.4 训练时长和成本估算

**计算公式**：

```python
# 总训练步数
total_steps = (样本数 // batch_size) × epochs

# 总 token 数
total_tokens = 样本数 × max_seq_len × epochs

# 训练时间估算（单卡 A100）
time_per_step ≈ 0.1-0.5 秒 (取决于模型大小和 batch_size)

total_time = total_steps × time_per_step
```

**示例计算**（MiniMind2-Small）：

```
假设条件:
- 数据量: 1M 样本
- batch_size: 32
- max_seq_len: 512
- epochs: 1
- GPU: RTX 3090

计算:
- 总步数: 1,000,000 / 32 = 31,250 steps
- 总 tokens: 1,000,000 × 512 = 512M tokens
- 每步时间: ~0.3 秒
- 总时间: 31,250 × 0.3 / 3600 ≈ 2.6 小时
```

**成本优化建议**：
1. 使用梯度累积增大等效 batch size
2. 使用混合精度训练（bfloat16）
3. 使用多卡 DDP 训练
4. 考虑使用更小的模型（26M）

## 3. 监督微调（SFT）

### 3.1 目的和作用

**SFT 的目标**：让模型学习对话能力，包括：
- 理解用户指令
- 生成合适的回复
- 遵循对话格式
- 特定任务适配

**预训练 vs SFT 对比**：

| 特性 | 预训练 | SFT |
|------|--------|-----|
| **数据** | 大规模无标注文本 | 高质量对话数据 |
| **目标** | 预测下一个 token | 生成合适的回复 |
| **Loss** | 全部计算 | 只在 assistant 回复处计算 |
| **学习率** | 较大 (5e-4) | 较小 (5e-7) |
| **数据量** | 很大 (B 级 tokens) | 较小 (M 级 tokens) |

### 3.2 数据格式要求

**ChatML 对话格式**：

```jsonl
{
  "conversations": [
    {"from": "human", "value": "你好"},
    {"from": "gpt", "value": "你好！有什么可以帮助你的？"},
    {"from": "human", "value": "介绍一下 Python"},
    {"from": "gpt", "value": "Python 是一种高级编程语言..."}
  ]
}
```

**转换为模型输入**：

```
<|im_start|>user
你好<|im_end|>
<|im_start|>assistant
你好！有什么可以帮助你的？<|im_end|>
<|im_start|>user
介绍一下 Python<|im_end|>
<|im_start|>assistant
Python 是一种高级编程语言...<|im_end|>
```

**Token ID 序列**：
```
[1, 123, 456, 2, 1, 789, 321, 111, 2, 1, 123, 456, 2, 1, 789, ...]
[BOS,user,user,EOS,BOS,asst,asst,asst,EOS,BOS,user,user,EOS,BOS,asst,...]
```

### 3.3 学习率调整策略

**SFT 使用更小的学习率**：

```python
# 预训练: lr = 5e-4
# SFT: lr = 5e-7 (小 1000 倍)
```

**为什么 SFT 学习率更小？**

```
预训练: 从随机初始化开始 → 需要大幅度调整权重 → 大学习率
            ↓
      预训练完成 (权重已较好)
            ↓
SFT: 微调预训练权重 → 只需小幅度调整 → 小学习率
```

**学习率对比表**：

| 阶段 | 学习率 | 原因 |
|------|--------|------|
| 预训练 | 5e-4 | 从头开始训练 |
| SFT | 5e-7 | 微调，避免破坏预训练权重 |
| LoRA | 1e-4 | 只训练新增的低秩矩阵 |
| DPO | 1e-6 | 需要非常精细的调整 |

### 3.4 损失掩码的实现

**动态损失掩码详解**：

```python
# dataset/lm_dataset.py:86-102
def _generate_loss_mask(self, input_ids):
    """生成动态损失掩码：只在 assistant 回复处计算 loss"""
    loss_mask = [0] * len(input_ids)
    i = 0
    while i < len(input_ids):
        # 查找 <|im_start|>assistant
        if input_ids[i:i + len(self.bos_id)] == self.bos_id:
            start = i + len(self.bos_id)
            end = start
            # 查找对应的 <|im_end|>
            while end < len(input_ids):
                if input_ids[end:end + len(self.eos_id)] == self.eos_id:
                    break
                end += 1
            # 标记 assistant 回复部分为 1
            for j in range(start + 1, min(end + len(self.eos_id) + 1, self.max_length)):
                loss_mask[j] = 1
            i = end + len(self.eos_id) if end < len(input_ids) else len(input_ids)
        else:
            i += 1
    return loss_mask
```

**损失掩码可视化**：

```
原始序列:
<|im_start|>user 你好<|im_end|><|im_start|>assistant 你好！<|im_end|>

Token IDs:
[1, 123, 456, 2, 1, 789, 321, 2]
 ↑        ↑     ↑  ↑        ↑  ↑
1        user  EOS 1      asst EOS

Loss Mask:
[0, 0,    0,    0, 0, 1,    1, 1]
不计算            ↑ 只在 assistant 部分计算

训练时:
user 输入 → loss_mask=0 → 不参与反向传播
assistant 回复 → loss_mask=1 → 参与反向传播
```

**为什么需要损失掩码？**

1. **User 输入**：是给定的，不需要模型预测
2. **Assistant 回复**：需要模型学习生成
3. **特殊 token**：`<|im_start|>` 和 `<|im_end|>` 是格式标记，不需要预测

## 4. LoRA 微调

### 4.1 参数高效微调原理

**全量微调 vs LoRA 对比**：

```
全量微调:
所有权重都参与训练 → 参数量大 = 模型参数量
26M 模型 → 26M 可训练参数

LoRA:
只训练低秩矩阵 → 参数量 = rank × (in + out)
26M 模型 → rank=8 → ~0.1M 可训练参数 (减少 99.6%)
```

**LoRA 数学原理**：

对于权重矩阵 W ∈ ℝ^(d×k)，LoRA 添加：
```
W' = W + ΔW = W + B × A
         ↑        ↑
      冻结    可训练
         (d×rank) × (rank×k)
```

**示例**：
```python
# 原始线性层: y = Wx, W ∈ ℝ^(512×512)
# LoRA 修改: y = Wx + BAx, B ∈ ℝ^(512×8), A ∈ ℝ^(8×512)
#
# 原始参数: 512 × 512 = 262,144
# LoRA 参数: 512 × 8 + 8 × 512 = 8,192
# 减少: 96.9%
```

### 4.2 LoRA 实现

**位置**：`model/model_lora.py`

```python
class LoRA(nn.Module):
    def __init__(self, in_features, out_features, rank=8):
        super().__init__()
        self.rank = rank
        # 低秩分解: W + BA
        self.A = nn.Linear(in_features, rank, bias=False)  # A 矩阵
        self.B = nn.Linear(rank, out_features, bias=False) # B 矩阵

        # 初始化
        nn.init.kaiming_uniform_(self.A.weight, a=math.sqrt(5))
        nn.init.zeros_(self.B.weight)  # B 初始化为 0，保证初始 ΔW=0

    def forward(self, x):
        # y = Wx + BAx
        return self.B(self.A(x))
```

**应用 LoRA 到模型**：

```python
def apply_lora_to_model(model, rank=8):
    """将 LoRA 应用到模型的线性层"""
    for name, module in model.named_modules():
        if isinstance(module, nn.Linear):
            # 只对方阵层应用 LoRA
            if module.in_features == module.out_features:
                # 创建 LoRA 层
                lora = LoRA(module.in_features, module.out_features, rank)
                # 替换原层
                parent_name = '.'.join(name.split('.')[:-1])
                child_name = name.split('.')[-1]
                parent = model.get_submodule(parent_name)
                setattr(parent, child_name, lora)

    return model
```

### 4.3 Rank 和 Alpha 参数

**Rank (秩)**：

| Rank | 参数量 | 效果 | 推荐场景 |
|------|--------|------|----------|
| 4 | 最少 | 效果一般 | 极度受限资源 |
| 8 | 较少 | 效果良好 | **推荐默认值** |
| 16 | 适中 | 效果更好 | 需要更好的适配 |
| 32 | 较多 | 效果接近全量 | 接近全量微调 |

**Alpha（缩放因子）**：

```python
# LoRA 权重计算
lora_weight = alpha / rank * B @ A
```

**典型配置**：
- `rank=8, alpha=16`：alpha/rank = 2
- `rank=16, alpha=32`：alpha/rank = 2
- alpha 通常是 rank 的 1-2 倍

### 4.4 何时使用 LoRA vs 全参数微调

**使用 LoRA 的场景**：
✅ 快速适配新任务
✅ 显存受限（< 8GB）
✅ 需要保存多个适配器
✅ 实验新想法

**使用全参数微调的场景**：
✅ 显存充足（> 24GB）
✅ 需要最好的效果
✅ 大规模任务适配
✅ 数据量很大（> 1M 样本）

## 5. DPO 强化学习

### 5.1 DPO vs PPO 的区别

**PPO (Proximal Policy Optimization)**：
```
奖励模型 → 奖励分数 → PPO 更新策略
  ↑                              ↓
需要训练                      计算复杂
```

**DPO (Direct Preference Optimization)**：
```
偏好数据 → 直接优化策略
  ↑            ↓
无需奖励模型   计算简单
```

**对比表**：

| 特性 | PPO | DPO |
|------|-----|-----|
| **需要奖励模型** | ✅ 是 | ❌ 否 |
| **训练复杂度** | 高 | 低 |
| **样本效率** | 中 | 高 |
| **稳定性** | 中 | 高 |
| **实现难度** | 难 | 易 |

### 5.2 双模型架构

**Policy 模型和 Reference 模型**：

```
┌─────────────────────────────────────────────────────────┐
│                    DPO 双模型架构                        │
└─────────────────────────────────────────────────────────┘

数据: (chosen, rejected) 对
           ↓
    ┌──────┴──────┐
    ↓             ↓
Policy Model    Reference Model
(可训练)         (冻结)
    ↓             ↓
log p_chosen    log p_chosen_ref
log p_rejected  log p_rejected_ref
    └──────┬──────┘
           ↓
      DPO Loss
           ↓
      更新 Policy
```

**为什么需要 Reference 模型？**

Reference 模型提供"基准"，防止 Policy 模型偏离太远：
```
如果只优化 Policy: 可能崩溃（只输出高概率 token）
加入 Reference: 约束 Policy 不要偏离 Reference 太远
```

### 5.3 DPO 损失函数推导

**Bradley-Terry 模型**：

```
P(y_chosen > y_rejected) = σ(β × (log p_chosen - log p_rejected))

其中:
- σ 是 sigmoid 函数
- β 是温度参数（控制偏好强度）
- log p 是对数概率
```

**DPO Loss**：

```python
# trainer/train_dpo.py (简化)
def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             reference_chosen_logps, reference_rejected_logps, beta=0.1):

    # 计算策略和参考模型的差异
    policy_logratios = policy_chosen_logps - policy_rejected_logps
    reference_logratios = reference_chosen_logps - reference_rejected_logps

    # DPO loss
    losses = -torch.log(
        torch.sigmoid(beta * (policy_logratios - reference_logratios))
    )

    return losses.mean()
```

**可视化 DPO Loss**：

```
Loss
│
│    当 policy_chosen >> policy_rejected
│   且 reference_chosen ≈ reference_rejected
│   时，Loss 最小
│
│   ／￣￣￣￣￣￣
│  ／
 │ ／
 │／
└─────────────────────→ policy_logratios - reference_logratios
```

### 5.4 Beta 参数的作用

**Beta (β) 控制偏好强度**：

| Beta | 效果 | 适用场景 |
|------|------|----------|
| 0.01 | 弱偏好，模型变化小 | 微调 |
| 0.1 | **适中偏好**（推荐） | 大多数场景 |
| 0.5 | 强偏好，模型变化大 | 需要显著改进 |
| 1.0 | 极强偏好 | 容易不稳定 |

**Beta 调优建议**：
```python
# 从小 beta 开始，逐渐增大
beta = 0.05  # 初始
# 观察效果，如果改进不明显则增大
beta = 0.1   # 推荐
# 如果模型不稳定则减小
beta = 0.05
```

## 6. 知识蒸馏

### 6.1 白盒蒸馏 vs 黑盒蒸馏

**白盒蒸馏**：

```
教师模型（可访问内部状态）
    ↓
学生模型（模仿教师输出+中间层）
    ↓
需要教师模型代码和权重
```

**黑盒蒸馏**：

```
教师模型（只能访问输出）
    ↓
学生模型（只模仿教师输出）
    ↓
只需要教师模型的 API
```

**MiniMind 的蒸馏**：

```python
# trainer/train_distillation.py
# 使用白盒蒸馏
teacher_loss = criterion(student_logits, teacher_logits)
student_loss = criterion(student_logits, labels)

# 联合损失
loss = α * teacher_loss + (1-α) * student_loss
```

### 6.2 蒸馏损失函数

**KL 散度损失**：

```python
def distillation_loss(student_logits, teacher_logits, temperature=3.0):
    """知识蒸馏损失（KL 散度）"""
    # 温度缩放
    student_log_probs = F.log_softmax(student_logits / temperature, dim=-1)
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)

    # KL 散度
    kl_div = F.kl_div(
        student_log_probs,
        teacher_probs,
        reduction='batchmean'
    )

    return kl_div * (temperature ** 2)
```

**温度 (Temperature) 的作用**：

```
温度 T = 1: 原始概率分布（尖锐）
温度 T > 1: 概率分布变平滑（ softer）

高温度的好处:
- 暴露教师模型的更多知识
- 学生模型学习更丰富的信息
- 避免过度自信
```

### 6.3 教师-学生模型配置

**配置示例**：

```python
# 教师模型: MiniMind2 (104M)
teacher_config = MiniMindConfig(
    hidden_size=768,
    num_hidden_layers=16,
    num_attention_heads=8
)

# 学生模型: MiniMind2-Small (26M)
student_config = MiniMindConfig(
    hidden_size=512,
    num_hidden_layers=8,
    num_attention_heads=8
```

**蒸馏训练**：

```python

    teacher_outputs.logits,
    temperature=3.0
)

    student_outputs.logits,
    labels
)

# 联合损失

```

## 7. 训练技巧汇总

### 7.1 混合精度训练（FP16/BF16）

**AMP (Automatic Mixed Precision)**：

```python
# trainer/train_pretrain.py:176
ctx = nullcontext() if device_type == "cpu" else torch.cuda.amp.autocast()

# 训练循环
with ctx:  # 自动混合精度
    res = model(X)
    loss = ...
```

**FP16 vs BF16 对比**：

| 特性 | FP16 | BF16 |
|------|------|------|
| **数值范围** | 窄 | 宽 |
| **精度** | 低（有时下溢） | 中 |
| **稳定性** | 需要梯度缩放 | 更稳定 |
| **推荐** | GPU 无 BF16 时 | GPU 支持 BF16 时 |

**梯度缩放（Scaler）**：

```python
scaler = torch.cuda.amp.GradScaler(enabled=(args.dtype in ['float16', 'bfloat16']))

# 前向和反向
with ctx:
    loss = model(X)

scaler.scale(loss).backward()  # 自动缩放梯度

# 优化器步进
scaler.step(optimizer)
scaler.update()
```

### 7.2 梯度累积和裁剪

**梯度累积**：

```python
# trainer/train_pretrain.py:58-69
loss = loss / args.accumulation_steps  # 先除以累积步数

scaler.scale(loss).backward()  # 反向传播

if (step + 1) % args.accumulation_steps == 0:
    # 累积完成，更新参数
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), args.grad_clip)
    scaler.step(optimizer)
    scaler.update()
    optimizer.zero_grad(set_to_none=True)
```

**等效 Batch Size**：
```
真实 batch_size = batch_size × accumulation_steps
例如: batch_size=32, accumulation_steps=8
等效 batch_size = 32 × 8 = 256
```

**梯度裁剪**：

```python
# 防止梯度爆炸
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

### 7.3 学习率调度

**Cosine Annealing**：

```python
def get_lr(current_step, total_steps, lr):
    """Cosine 退火学习率"""
    return lr / 10 + 0.5 * lr * (1 + math.cos(math.pi * current_step / total_steps))
```

**Warmup（可选）**：

```python
def get_lr_with_warmup(current_step, total_steps, lr, warmup_steps):
    """带 warmup 的学习率"""
    if current_step < warmup_steps:
        # 线性 warmup
        return lr * current_step / warmup_steps
    else:
        # Cosine 退火
        progress = (current_step - warmup_steps) / (total_steps - warmup_steps)
        return lr / 10 + 0.5 * lr * (1 + math.cos(math.pi * progress))
```
