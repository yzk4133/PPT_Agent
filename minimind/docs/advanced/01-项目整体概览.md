## 1. 项目背景

### 1.1 MiniMind 的设计目标

MiniMind 是一个**轻量级、可扩展、教学友好**的大语言模型训练框架，主要目标包括：

- **极致轻量**：从 26M 到 145M 参数的超小模型，适合在消费级 GPU 上训练和部署
- **完整流程**：涵盖从预训练到 DPO 的完整训练流程，展示 LLM 训练的全生命周期
- **纯净实现**：核心代码完全自研，不依赖复杂的第三方库，便于学习和理解
- **生产就绪**：支持 LoRA、DPO、MoE 等先进技术，提供 OpenAI 兼容的 API 服务

### 1.2 与其他开源项目的对比

| 特性 | MiniMind | Llama 3 | Qwen 2.5 | ChatGLM |
|------|----------|---------|----------|---------|
| **最小参数量** | 26M | 8B | 0.5B | 6B |
| **训练门槛** | 单张消费级 GPU | 多张 A100 | 多张 A100 | 多张 A100 |
| **代码可读性** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| **教学友好** | ✅ 是 | ❌ 否 | ❌ 否 | ❌ 否 |
| **生产环境** | 可用 | ✅ 成熟 | ✅ 成熟 | ✅ 成熟 |

**适用场景对比**：
- **MiniMind**：学习研究、边缘设备部署、快速验证想法
- **Llama/Qwen**：生产环境、复杂任务、需要强大性能
- **ChatGLM**：中文场景、金融领域应用

### 1.3 适用场景和限制

#### 适用场景
✅ **学习 LLM 原理**：代码简洁清晰，适合深度学习
✅ **算法研究**：快速验证新想法（如 MoE、新的注意力机制）
✅ **边缘部署**：模型极小，可部署在手机、树莓派等设备
✅ **意图识别/分类**：小任务场景，效果良好
✅ **教学演示**：课堂、培训等需要快速演示的场景

#### 限制
❌ **复杂推理**：参数量小，不适合复杂逻辑推理
❌ **长文本生成**：上下文理解能力有限
❌ **多语言**：主要针对中文优化
❌ **知识密集**：训练数据有限，知识储备不足

## 2. 技术架构总览

### 2.1 模型规模对比

MiniMind 提供三种不同规模的模型配置：

| 模型 | 参数量 | hidden_size | layers | heads | kv_heads | 适用场景 |
|------|--------|-------------|--------|-------|----------|----------|
| **MiniMind2-Small** | 26M | 512 | 8 | 8 | 2 | 快速验证、学习 |
| **MiniMind2** | 104M | 768 | 16 | 8 | 2 | 一般任务 |
| **MiniMind2-MoE** | 145M | 640 | 8 | 8 | 2 | 需要专家路由 |

**参数量计算公式**（以 MiniMind2-Small 为例）：
```
Embedding 层: vocab_size × hidden_size = 6400 × 512 = 3.3M
单层 Transformer:
  - Attention: 4 × hidden_size² = 1.0M
  - FFN: 8 × hidden_size² = 2.1M
  - 共计约 3.1M
8 层 Transformer: 3.1M × 8 = 24.8M
总计约 26M 参数
```

### 核心文件说明

#### model/model_minimind.py (核心文件)
- **MiniMindConfig** (lines 8-68)：模型配置类，定义所有超参数
- **RMSNorm** (lines 84-94)：归一化层，比 LayerNorm 更高效
- **RoPE** (lines 97-118)：旋转位置编码，支持长文本外推
- **Attention** (lines 133-227)：GQA 注意力机制，支持 KV Cache
- **FeedForward** (lines 230-251)：SwiGLU 激活的前馈网络
- **MOEFeedForward** (lines 254-387)：混合专家模型
- **MiniMindBlock** (lines 390-425)：Transformer 单层
- **MiniMindForCausalLM** (lines 496-536)：完整因果语言模型

#### dataset/lm_dataset.py
- **PretrainDataset** (lines 16-51)：处理预训练数据
- **SFTDataset** (lines 54-119)：处理对话数据，支持 ChatML 格式
- **DPODataset** (lines 122-195)：处理偏好数据，对比 chosen 和 rejected

#### trainer/*.py
所有训练脚本共享相似的结构：
- 命令行参数解析（argparse）
- 数据加载（DataLoader + DistributedSampler）
- 学习率调度（Cosine Annealing）
- 混合精度训练（AMP）
- 分布式训练（DDP）
- 模型保存和日志记录

## 4. 快速开始

### 4.1 环境配置

```bash

git clone https://github.com/jijiajian/MiniMind.git
cd MiniMind

# 2. 创建虚拟环境

# 或
```

### 4.3 训练命令速查

#### 预训练（从头开始）
```bash
# 最小模型 26M，单 GPU 训练
python trainer/train_pretrain.py \
    --data_path data/pretrain_hq.jsonl \
    --out_dir out \
    --hidden_size 512 \
    --num_hidden_layers 8 \
    --epochs 1 \
    --batch_size 32 \
    --learning_rate 5e-4 \
    --max_seq_len 512 \
    --device cuda:0

# 多 GPU 训练（DDP）
torchrun --nproc_per_node 2 trainer/train_pretrain.py \
    --data_path data/pretrain_hq.jsonl \
    --ddp \
    ...
```

#### 监督微调（SFT）
```bash
# 加载预训练权重进行 SFT
python trainer/train_full_sft.py \
    --data_path data/sft_mini_512.jsonl \
    --out_dir out \
    --hidden_size 512 \
    --num_hidden_layers 8 \
    --epochs 2 \
    --batch_size 16 \
    --learning_rate 5e-7 \
    --max_seq_len 512 \
    --device cuda:0
```

#### LoRA 微调
```bash
python trainer/train_lora.py \
    --pretrained_model_path out/full_sft_512.pth \
    --data_path data/sft_mini_512.jsonl \
    --out_dir out \
    --lora_rank 8 \
    --epochs 3 \
    --batch_size 32 \
    --learning_rate 1e-4
```

#### DPO 训练
```bash
python trainer/train_dpo.py \
    --pretrained_model out/full_sft_512.pth \
    --data_path data/dpo_data.jsonl \
    --out_dir out \
    --beta 0.1 \
    --epochs 1 \
    --batch_size 8
```

## 5. 常见问题速查

### Q1: 显存不足怎么办？
**A**: 有几种优化方案：
1. 减小 `batch_size`
2. 开启梯度累积：`--accumulation_steps 8`
3. 使用更小的模型：`--hidden_size 512`
4. 启用混合精度：`--dtype bfloat16`

### Q2: 训练速度慢？
**A**: 检查以下几点：
1. 确保 Flash Attention 可用：`args.flash_attn = True`
2. 使用 DDP 多卡训练：`--ddp`
3. 增加 `num_workers`（数据加载）
4. 使用 SSD 存储数据

### Q3: Loss 不收敛？
**A**: 常见原因：
1. 学习率过大：降低 `learning_rate`
2. 数据质量差：检查数据格式
3. 模型配置错误：确认 `vocab_size` 和 tokenizer 匹配

### Q4: 如何选择模型规模？
**A**: 根据任务和硬件：
- **学习/验证**：MiniMind2-Small (26M)
- **一般任务**：MiniMind2 (104M)
- **需要专家路由**：MiniMind2-MoE (145M)
