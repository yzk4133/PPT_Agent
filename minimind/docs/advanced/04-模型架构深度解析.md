## 1. 整体架构设计

### 1.1 Transformer Decoder 结构

**MiniMind 采用 Decoder-only 架构**，与 GPT、LLaMA 相同。

```
输入 (Token IDs)
    ↓
Embedding 层 (词嵌入)
    ↓
┌─────────────────────────┐
│   Transformer Block × N │
│   ┌───────────────────┐ │
│   │ RMSNorm           │ │
│   │ Self-Attention    │ │ ← 使用 RoPE + GQA
│   │ Residual Add      │ │
│   │ RMSNorm           │ │
│   │ FeedForward/MoE   │ │ ← SwiGLU / MoE
│   │ Residual Add      │ │
│   └───────────────────┘ │
└─────────────────────────┘
    ↓
Final RMSNorm
    ↓
LM Head (输出词表概率)
```

## 2. 核心组件详解

### 2.1 RMSNorm

**位置**：`model/model_minimind.py:84-94`

**公式**：
```
RMSNorm(x) = x / RMS(x) × g
RMS(x) = √(mean(x²) + ε)
```

**与 LayerNorm 对比**：

| 特性 | LayerNorm | RMSNorm |
|------|-----------|---------|
| 中心化 | 减去均值 | 无 |
| 计算复杂度 | O(d) | O(d) |
| 数值稳定性 | 更高 | 略低 |
| 推理速度 | 慢 | **快 10-20%** |

**代码实现**：
```python
class RMSNorm(torch.nn.Module):
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def _norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        return self.weight * self._norm(x.float()).type_as(x)
```

### 2.2 RoPE 位置编码

**位置**：`model/model_minimind.py:97-118`

**核心思想**：通过旋转矩阵编码相对位置

**数学原理**：
```
对于位置 m 的向量 q：
q_m = rotate(q, mθ)

旋转后的内积包含相对位置信息：
⟨q_m, k_n⟩ = f(q, k, n - m)
```

**优势**：
1. **相对位置**：自然编码相对距离
2. **外推能力**：可泛化到训练长度之外
3. **无额外参数**：不增加模型参数

**实现**：
```python
def precompute_freqs_cis(dim: int, end: int, theta: float = 1e6):
    # 预计算频率
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(end, device=freqs.device)
    freqs = torch.outer(t, freqs).float()
    freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1)
    freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1)
    return freqs_cos, freqs_sin
```

### 2.4 FeedForward / MoE

**标准 FFN**：`model/model_minimind.py:230-251`

```python
class FeedForward(nn.Module):
    def __init__(self, config):
        intermediate_size = int(config.hidden_size * 8 / 3)  # SwiGLU 标准
        self.gate_proj = nn.Linear(hidden_size, intermediate_size)
        self.up_proj = nn.Linear(hidden_size, intermediate_size)
        self.down_proj = nn.Linear(intermediate_size, hidden_size)

    def forward(self, x):
        # SwiGLU 激活
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

**SwiGLU 激活**：
```
SwiGLU(x) = Swish(xW) ⊙ (xV)
其中 Swish(x) = x × sigmoid(x)
```

**MoE**：`model/model_minimind.py:254-387`

```python
class MOEFeedForward(nn.Module):
    def __init__(self, config):
        # 路由专家
        self.experts = nn.ModuleList([FeedForward(config) for _ in range(n_routed_experts)])
        self.gate = MoEGate(config)  # 门控网络
        # 共享专家
        self.shared_experts = nn.ModuleList([...])

    def forward(self, x):
        # 1. 门控选择专家
        topk_idx, topk_weight, aux_loss = self.gate(x)

        # 2. 分发到各专家
        for i, expert in enumerate(self.experts):
            y[flat_topk_idx == i] = expert(x[flat_topk_idx == i])

        # 3. 加权组合
        y = (y * topk_weight.unsqueeze(-1)).sum(dim=1)

        # 4. 加上共享专家
        for expert in self.shared_experts:
            y = y + expert(identity)
```

## 3. 关键技术点

### 3.1 Flash Attention

**启用条件**：
```python
self.flash = (
    hasattr(torch.nn.functional, "scaled_dot_product_attention")  # PyTorch ≥ 2.0
    and args.flash_attn
)

if use_flash:
    output = F.scaled_dot_product_attention(
        xq, xk, xv,
        attn_mask=None,
        is_causal=True  # 因果掩码
    )
```

**优势**：
- **内存高效**：O(N) 而非 O(N²)
- **速度快**：融合 kernel，减少 H2H 内存访问

### 3.2 MoE 门控机制

**位置**：`model/model_minimind.py:254-322`

**Top-k 门控**：
```python
# 1. 计算每个专家的得分
logits = F.linear(hidden_states, self.weight)

# 2. Softmax 归一化
scores = logits.softmax(dim=-1)

# 3. 选择 top-k 专家
topk_weight, topk_idx = torch.topk(scores, k=self.top_k, dim=-1)

# 4. 归一化权重
if self.top_k > 1:
    denominator = topk_weight.sum(dim=-1, keepdim=True) + 1e-20
    topk_weight = topk_weight / denominator
```

**负载均衡辅助损失**：
```python
# 鼓励均匀使用各专家
aux_loss = (Pi * fi).sum() * alpha
```

### 3.3 权重共享

**Embedding 和 LM Head 共享**：

## 4. 模型配置参数

### 4.1 核心参数表

**MiniMind2-Small (26M)**：
```python
hidden_size = 512
num_hidden_layers = 8
num_attention_heads = 8
num_key_value_heads = 2
vocab_size = 6400
max_position_embeddings = 32768
intermediate_size = 1408  # 512 * 8 / 3, 向上取整到 64 倍数
```

**参数量计算**：
```
Embedding: 6400 × 512 = 3.3M
Attention (单层): 4 × 512² = 1.0M
FFN (单层): 8 × 512² ≈ 2.1M
8 层 Transformer: 3.1M × 8 = 24.8M
总计: ≈ 26M
```

### 4.2 不同规模对比

| 模型 | hidden_size | layers | heads | 参数量 |
|------|-------------|--------|-------|--------|
| MiniMind2-Small | 512 | 8 | 8/2 | 26M |
| MiniMind2 | 768 | 16 | 8/2 | 104M |
| MiniMind2-MoE | 640 | 8 | 8/2 | 145M |

### 4.3 参数调优指南

**hidden_size**：
- **影响**：模型表达能力、参数量
- **选择**：2 的幂次（256, 512, 768, 1024）
- **权衡**：越大越强，但训练慢

**num_hidden_layers**：
- **影响**：模型深度、能力
- **选择**：通常 6-24 层
- **权衡**：越深越强，但易梯度消失

**num_attention_heads**：
- **影响**：注意力多样性
- **选择**：能被 hidden_size 整除
- **关系**：head_dim = hidden_size / num_heads

## 5. 设计权衡

### 5.1 深度 vs 宽度

**深度优先**（MiniMind2）：
- 16 层 × 768 维
- 优势：更强的特征抽象
- 劣势：训练难度高

**宽度优先**（MiniMind2-Small）：
- 8 层 × 512 维
- 优势：训练快、易部署
- 劣势：能力有限

**经验**：小模型优先增加宽度，大模型优先增加深度

### 5.2 词表大小的影响

**小词表（6400）**：
- ✅ Embedding 参数少
- ✅ 训练快
- ❌ OOV 多

**大词表（32000）**：
- ✅ 覆盖率高
- ❌ 参数量大
- ❌ 训练慢

**MiniMind 选择 6400**：适合小模型

### 5.3 MoE vs Dense 模型

**Dense 模型**：
- 所有参数参与计算
- 训练稳定
- 推理慢（参数量大）

**MoE 模型**：
- 只激活部分专家
- 训练复杂（需要辅助损失）
- 推理快（稀疏激活）

**选择建议**：
- 任务简单 → Dense
- 任务复杂、资源充足 → MoE

## 7. 总结

### 核心要点

1. **Decoder-only**：适合生成任务
2. **RoPE**：相对位置编码，可外推
3. **GQA**：平衡效率和性能
4. **SwiGLU**：门控激活，效果好
5. **MoE**：参数高效，但训练复杂
6. **权重共享**：Embedding 和 LM Head 共享

### 设计原则

1. **简洁优先**：避免过度设计
2. **性能权衡**：效率和效果平衡
3. **可扩展性**：易于增加规模
