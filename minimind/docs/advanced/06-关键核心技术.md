## 1. 位置编码技术

### 1.1 RoPE 原理和实现

**RoPE (Rotary Position Embedding)**：旋转位置编码

**核心思想**：通过旋转矩阵编码相对位置信息

**数学推导**：

对于位置 m 的查询向量 q：
```
q_m = rotate(q, mθ)
```

旋转矩阵：
```
R(θ, m) = [
  [cos(mθ), -sin(mθ)],
  [sin(mθ),  cos(mθ)]
]
```

**关键性质**：
```
⟨q_m, k_n⟩ = ⟨rotate(q, mθ), rotate(k, nθ)⟩
            = f(q, k, n - m)  # 只依赖相对位置
```

**MiniMind 实现**：`model/model_minimind.py:97-118`

```python
def precompute_freqs_cis(dim: int, end: int, theta: float = 1e6):
    # 预计算频率
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(end, device=freqs.device)
    freqs = torch.outer(t, freqs).float()

    # 计算 cos 和 sin
    freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1)
    freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1)
    return freqs_cos, freqs_sin
```

### 1.2 绝对位置编码 vs 相对位置编码

**绝对位置编码（BERT）**：
```python
# 每个位置有固定的 embedding
pos_embed = nn.Embedding(max_len, d_model)
x = x + pos_embed(positions)
```

**问题**：无法泛化到训练长度之外

**相对位置编码（RoPE）**：
```python
# 通过旋转编码相对位置
xq, xk = apply_rotary_pos_emb(xq, xk, cos, sin)
```

**优势**：
- ✅ 自然编码相对距离
- ✅ 可以外推到更长序列
- ✅ 无额外参数

### 1.3 长度外推能力

**外推**：模型处理比训练时更长序列的能力

**RoPE 外推原理**：
```
训练长度：512
测试长度：1024

RoPE 通过旋转角度自然处理更长的位置
```

**实际效果**：
- MiniMind：`max_position_embeddings=32768`
- 训练时可以较短（如 512），推理时可以更长

## 2. 注意力机制演进

### 2.1 Multi-Head Attention → GQA → MQA

**MHA (Multi-Head Attention)**：
```
每个 head 独立的 Q、K、V
参数量大：3 × num_heads × d_model²
```

**MQA (Multi-Query Attention)**：
```
所有 head 共享一组 K、V
参数量小：d_model × d_model + 2 × d_k × d_model
问题：性能下降
```

**GQA (Grouped Query Attention)**：
```
head 分组共享 K、V
折中方案：参数和性能平衡
MiniMind 采用 GQA
```

**MiniMind 配置**：
```python
num_attention_heads = 8        # Q head 数
num_key_value_heads = 2        # KV head 数
# 每 4 个 Q head 共享 1 组 KV
```

**对比**：
| 方案 | Q heads | KV heads | 参数量 | 性能 |
|------|---------|----------|--------|------|
| MHA | 8 | 8 | 100% | 100% |
| GQA | 8 | 2 | ~70% | ~98% |
| MQA | 8 | 1 | ~60% | ~95% |

### 2.2 KV Cache 的作用和优化

**KV Cache**：缓存历史 Key-Value 向量

**作用**：
```
首次：计算所有 token 的 K、V
后续：只计算新 token 的 K、V，复用缓存的 K、V

加速比：2-3 倍
```

**实现**：`model/model_minimind.py:184-188`

```python
# kv_cache 实现
if past_key_value is not None:
    xk = torch.cat([past_key_value[0], xk], dim=1)
    xv = torch.cat([past_key_value[1], xv], dim=1)

past_kv = (xk, xv) if use_cache else None
```

**显存占用**：
```
batch_size × num_layers × 2 × num_heads × seq_len × head_dim × 2 bytes
```

### 2.3 Flash Attention 的原理

**Flash Attention**：优化的注意力算法

**核心优化**：
1. **分块计算**：将长序列分成小块
2. **融合 kernel**：减少 H2H 内存访问
3. **在线 Softmax**：避免存储完整的注意力矩阵

**内存复杂度**：
```
标准 Attention: O(N²)
Flash Attention: O(N)
```

**启用条件**：
```python
self.flash = (
    hasattr(torch.nn.functional, "scaled_dot_product_attention")
    and args.flash_attn
)
```

## 3. 归一化技术

### 3.1 LayerNorm vs RMSNorm

**LayerNorm**：
```
LN(x) = γ × (x - μ) / √(σ² + ε) + β
```

**RMSNorm**：
```
RMSNorm(x) = γ × x / √(mean(x²) + ε)
```

**关键区别**：
1. **RMSNorm 不中心化**：不减去均值
2. **计算更简单**：减少一次均值计算
3. **速度更快**：推理快 10-20%

**性能对比**：
| 特性 | LayerNorm | RMSNorm |
|------|-----------|---------|
| 计算量 | 2 × mean | 1 × mean |
| 数值稳定性 | 更高 | 略低 |
| 推理速度 | 基准 | **快 10-20%** |

**MiniMind 选择 RMSNorm**：速度优势明显

### 3.2 Pre-Norm vs Post-Norm

**Post-Norm（原始 Transformer）**：
```
x = x + Attention(LayerNorm(x))
x = x + FFN(LayerNorm(x))
```

**问题**：训练不稳定，深层网络难训练

**Pre-Norm（现代架构）**：
```
x = x + Attention(LayerNorm(x))  # Norm 在 Attention 前
x = x + FFN(LayerNorm(x))        # Norm 在 FFN 前
```

**优势**：
- ✅ 训练更稳定
- ✅ 梯度流动更顺畅
- ✅ 易于训练深层网络

**MiniMind 使用 Pre-Norm**

### 3.3 归一化位置的影响

**Pre-Norm 结构**：
```
RMSNorm(x) → Attention → Add
RMSNorm(x) → FFN → Add
```

**好处**：
1. 残差连接前归一化，梯度更稳定
2. 每层的输入分布更一致

## 4. 激活函数

### 4.1 ReLU vs SwiGLU

**ReLU**：
```
ReLU(x) = max(0, x)
```

**SwiGLU**：
```
SwiGLU(x) = Swish(xW) ⊙ (xV)
Swish(x) = x × sigmoid(x)
```

**SwiGLU 优势**：
1. **门控机制**：更丰富的表达能力
2. **平滑梯度**：避免梯度消失
3. **性能更好**：实验验证有效

**MiniMind 使用 SwiGLU**

### 4.2 门控线性单元的实现

**实现**：`model/model_minimind.py:230-251`

```python
class FeedForward(nn.Module):
    def forward(self, x):
        # gate_proj: x → W_gate → Swish
        # up_proj: x → W_up
        # down_proj: (Swish(xW_gate) ⊙ xW_up) → W_down
        return self.down_proj(
            self.act_fn(self.gate_proj(x)) * self.up_proj(x)
        )
```

**中间层大小**：
```python
intermediate_size = int(hidden_size * 8 / 3)
# SwiGLU 标准配置
```

### 4.3 对训练稳定性的影响

**SwiGLU 的好处**：
1. **平滑输出**：避免硬截断
2. **门控调节**：动态控制信息流
3. **梯度平滑**：训练更稳定

## 5. MoE 技术

### 5.1 MoE 的基本原理

**MoE (Mixture of Experts)**：混合专家模型

**思想**：
```
每个 token 通过门控选择部分专家
不同专家学习不同的模式
```

**结构**：
```
输入 → 门控网络 → Top-k 选择 → 专家处理 → 加权组合
```

### 5.2 门控机制设计

**Top-k 门控**：`model/model_minimind.py:254-322`

```python
# 1. 计算专家得分
logits = F.linear(hidden_states, self.weight)  # [bsz*seq, n_experts]

# 2. Softmax 归一化
scores = logits.softmax(dim=-1)

# 3. 选择 top-k 专家
topk_weight, topk_idx = torch.topk(scores, k=2, dim=-1)

# 4. 归一化权重
topk_weight = topk_weight / topk_weight.sum(dim=-1, keepdim=True)
```

**参数**：
```python
n_routed_experts = 4      # 总专家数
num_experts_per_tok = 2   # 每个 token 选 2 个
```

### 5.3 负载均衡策略

**问题**：某些专家过载，某些专家闲置

**解决方案**：辅助损失

**实现**：
```python
# 计算每个专家的使用频率
ce = mask_ce.float().mean(0)  # [n_experts]

# 计算期望频率
Pi = scores_for_aux.mean(0)   # [n_experts]

# 辅助损失：鼓励均匀分布
aux_loss = (Pi * fi).sum() * alpha
```

**总损失**：
```
Total Loss = Main Loss + α × Aux Loss
                     (0.1)
```

### 5.4 训练稳定性问题

**MoE 训练挑战**：
1. **负载不均衡**：通过辅助损失解决
2. **梯度爆炸**：梯度裁剪
3. **通信开销**：分布式训练时优化

**MiniMind 解决方案**：
```python
# 1. 辅助损失
aux_loss_alpha = 0.1

# 2. 梯度裁剪
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 3. 共享专家
n_shared_experts = 1  # 确保有基础能力
```

## 6. 模型优化技术

### 6.1 权重共享

**Embedding 和 LM Head 共享**：

```python
# model/model_minimind.py:506
self.model.embed_tokens.weight = self.lm_head.weight
```

**好处**：
1. **减少参数**：节省 3.3M 参数
2. **正则化**：绑定输入输出空间
3. **提升效果**：实验验证有效

### 6.2 梯度检查点（Gradient Checkpointing）

**目的**：用计算换显存

**原理**：
```
前向传播时：不保存中间激活
反向传播时：重新计算中间激活

显存节省：30-50%
计算增加：20-30%
```

**适用场景**：
- 显存不足
- 模型很大

### 6.3 参数量化

**量化方法**：
1. **FP16/BF16**：训练时混合精度
2. **INT8**：推理时量化
3. **INT4**：极端压缩

**实现**：
```python
# 动态量化
quantized_model = torch.quantization.quantize_dynamic(
    model,
    {torch.nn.Linear},
    dtype=torch.qint8
)
```

**效果**：
- 模型大小减半
- 推理速度提升 2-3 倍
- 精度损失 < 1%
